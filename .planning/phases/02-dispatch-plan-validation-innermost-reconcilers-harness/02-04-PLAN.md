CLI:
- `stub-subagent --envelope /workspace/envelopes/$TIDE_TASK_UID/in.json`
- Or, if `--envelope` omitted, read from env var `TIDE_ENVELOPE_PATH`; default `/workspace/envelopes/$TIDE_TASK_UID/in.json`.
- Output path derived from input path: replace `in.json` with `out.json` in the same directory.

Test modes (Dev.TestMode enum, D-F3):
- `success`: read declared output paths from EnvelopeIn; write a canned "result.txt" with content "stub success" under the FIRST declared output path; write `out.json` with `{ExitCode: 0, Result: "success", Reason: "stub testMode=success", Usage: {InputTokens: 100, OutputTokens: 200, EstimatedCostCents: 1, Iterations: 1}, Artifacts: [<the relative path of result.txt>], CompletedAt: time.Now()}`; exit 0.
- `fail-exit-1`: write out.json with `{ExitCode: 1, Result: "forced-failure", Reason: "stub testMode=fail-exit-1", Usage: {InputTokens: 0, OutputTokens: 0, EstimatedCostCents: 0, Iterations: 0}, CompletedAt: time.Now()}`; exit 1.
- `hang`: do NOT write out.json; loop with `time.Sleep(time.Hour)` after installing a SIGTERM handler that exits 0 immediately on signal (so harness SIGTERM produces a clean kill rather than a runaway). Per RESEARCH.md Open Question #6: `time.Sleep(time.Hour)` makes the test deterministic regardless of cap value.
- `exceed-output-paths`: write `/workspace/escape/leak.txt` with content "leaked" (this path is intentionally OUTSIDE EnvelopeIn.DeclaredOutputPaths so Plan 06's HARN-05 validator can reject it); THEN write out.json with `{ExitCode: 0, Result: "success", Reason: "stub testMode=exceed-output-paths", Usage: {InputTokens: 100, OutputTokens: 100, EstimatedCostCents: 1, Iterations: 1}, Artifacts: ["/workspace/escape/leak.txt"], CompletedAt: time.Now()}`; exit 0.
- Default (Dev.TestMode == "" or Dev == nil): treat as `success`.

Validation:
- On envelope load, call `pkgdispatch.ValidateAPIVersionKind(env.APIVersion, env.Kind, pkgdispatch.KindTaskEnvelopeIn)` (from Plan 01); on error, write a `result: invalid-envelope` out.json (if writeable) and exit 2. Exit 2 distinguishes "envelope malformed" from "task failed" (exit 1) and "success" (exit 0).

Image:
- Multi-stage Dockerfile mirroring the kubebuilder-scaffolded root `Dockerfile`:
  - Stage 1: `golang:1.26-alpine` builder; `CGO_ENABLED=0 GOOS=linux go build -ldflags="-s -w" -o /out/stub-subagent ./cmd/stub-subagent`.
  - Stage 2: `gcr.io/distroless/static:nonroot` (or `scratch` if no CA bundle needed; since stub never makes outbound calls, scratch is fine — planner picks).
  - `USER 1000` (matches D-G3 subagent UID).
  - `ENTRYPOINT ["/usr/local/bin/stub-subagent"]`.
- Tag image as `ghcr.io/jsquirrelz/tide-stub-subagent:v0.1.0-dev`. Plan 13's kind suite tags `:test` separately.
