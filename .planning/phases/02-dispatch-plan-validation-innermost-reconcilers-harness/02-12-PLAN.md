| make helm CI gate → committed charts/ tree | `git diff --quiet charts/` catches any drift between source-of-truth (`hack/helm/`) and the published chart. Prevents hand-edited drift. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-12-01 | Information Disclosure | Signing key value visible in `kubectl describe secret` | accept | RBAC controls who can describe Secrets; operators with that permission already see the key. Standard K8s posture. |
| T-02-12-02 | Tampering | helm upgrade replaces signing key → in-flight tokens fail HMAC verify | mitigate | `resource-policy: keep` + `lookup` guard: upgrade leaves existing Secret intact. In-flight tokens stay valid. |
| T-02-12-03 | Denial of Service | Reinstall without resource-policy: keep → key changes → all running Tasks fail | mitigate | The annotation IS the defense; documented in chart README + tested via the lookup test. |
| T-02-12-04 | Spoofing | A Subagent SA gaining unexpected RBAC verbs (D-A4 violation) | mitigate | `tide-subagent` SA template ships with NO RoleBinding; CRD-06 carry-forward gate (`make verify-no-rbac-wildcards`) catches accidental Role attachment. |
| T-02-12-05 | Tampering | Augment-script drift → chart layout changes silently | mitigate | CI gate `git diff --quiet charts/` post-`make helm`. PATTERNS.md preserves the Phase 1 idempotency contract. |
| T-02-12-06 | Information Disclosure | TIDE_SIGNING_KEY env var visible in `kubectl describe deploy` | accept | Standard K8s Deployment env declaration; same posture as ANTHROPIC_API_KEY (which is also envFrom). RBAC controls who can describe deploys. |
