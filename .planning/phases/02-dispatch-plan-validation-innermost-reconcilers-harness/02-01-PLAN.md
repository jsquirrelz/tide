<objective>
Land the public Go contract that every other Phase 2 deliverable depends on. The `pkg/dispatch` package becomes the **out-of-tree** API contract for subagent image authors per D-A1 — they import this package, decode the input envelope, and produce the output envelope. Output: `pkg/dispatch/` leaf package (stdlib + JSON only, no Apache header, leaf-style mirroring `pkg/dag`), plus the `verify-dispatch-imports` Makefile gate extending the DAG-05 firewall to cover `pkg/dispatch`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| controller → subagent (in.json) | Orchestrator writes EnvelopeIn JSON on PVC; subagent process reads it. Boundary crossing for prompt, signed-token, declared-output-paths. |
| subagent → controller (out.json) | Harness writes EnvelopeOut JSON on PVC; controller reads it on Job completion. Boundary crossing for usage tally, exit reason. |
| external image author → pkg/dispatch | Out-of-tree subagent image authors import `github.com/jsquirrelz/tide/pkg/dispatch`. Boundary is the public Go API surface. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Tampering | EnvelopeIn JSON on PVC | mitigate | apiVersion + kind discriminator (D-A3) gives version pinning; harness rejects unknown apiVersion via `ValidateAPIVersionKind` before consuming any field; Plan 06 enforces. |
| T-02-01-02 | Information Disclosure | Out-of-tree consumers seeing fields they shouldn't depend on | mitigate | Dev field is `*Dev` (pointer + omitempty) so production envelopes don't leak the field; doc-comment on Dev explicitly says "real Claude-backed image MUST ignore this struct entirely" per RESEARCH.md Pitfall 9. |
| T-02-01-03 | Elevation of Privilege | Subagent image author importing internal/ packages by mistake | mitigate | `verify-dispatch-imports` Makefile gate forbids any internal/* / k8s.io/* / anthropic-sdk-go imports under pkg/dispatch transitive closure (DAG-05 mirror). |
| T-02-01-04 | Denial of Service | Unbounded envelope size | accept | Envelopes are JSON files on PVC — size bounded by filesystem quota at the PVC layer (not at the type layer). Pitfall 17 cardinality discipline applies to metrics, not envelope payload; documented as v1.x consideration. |
| T-02-01-05 | Repudiation | Envelope shape change breaks external image authors silently | mitigate | apiVersion field is the contract version; bumping to v1beta1 is the breaking-change path (per RESEARCH.md "JSON tag stability is the public contract"); v1alpha1 freezes after this plan ships. |
