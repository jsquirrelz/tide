<objective>
Arm tide-push's existing artifact-staging machinery for planning-artifact commits: a new `--stage-envelopes` flag that maps envelope directories (UID-keyed on the PVC) to human-readable `.tide/planning/<kind>/<name>/` destinations on the run branch. The existing `--artifact-paths` flag stages src and dst at the SAME relative path, which would put UID-keyed envelope paths on the branch — violating D-02's human-readable requirement — so a src→dst mapping flag is required (RESEARCH Pattern 1).

This implements the write half of DASH-02 under its superseded wording (Pitfall 10 — the PLAN carries the replacement, the verifier must NOT test ConfigMaps):

> **DASH-02 (reworded per D-01/D-03):** The level's planning `*.md` + `children/*.json` are committed to the run branch under `.tide/planning/<kind>/<name>/` at planner completion, via the tide-push staging pipeline. No truncation or size-capping anywhere in the pipeline — size problems fail loudly (typed envelope reason / condition), never silently trim. PVC and git remain the source of truth. The old owner-ref'd-ConfigMap/CR-deletion-GC clause is retired with no analogue — artifacts deliberately persist on the run branch as the durable decision record (D-02).

**PRECONDITION:** Phases 34 (rewritten push internals: single-flight, verify-in-push, lastPushedSHA) and 36 (agentSignature identity) are merged to main before this plan executes. The executor MUST re-read cmd/tide-push/main.go at execution start — the pattern-map excerpts reflect the pre-34 shape; anchor on function/flag names, not line numbers (RESEARCH "Dependency correction").

Purpose: DASH-02 write path — planning artifacts land on the run branch with full fidelity.
