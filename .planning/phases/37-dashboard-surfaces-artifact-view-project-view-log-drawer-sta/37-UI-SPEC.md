# Phase 37 — UI Design Contract

> Visual and interaction contract for the three Phase 37 surfaces: the Planning-DAG artifact viewer (DASH-01/02), the project settings view (DASH-03), and honest log-drawer states (DASH-04). Generated by gsd-ui-researcher; verified by gsd-ui-checker.

> **Audience (unchanged from 04-UI-SPEC):** Kubernetes operators who use `kubectl`, `k9s`, `helm`, `argocd`. Dense, status-first, monospace-friendly. Aesthetic peers: Argo CD UI, k9s, Headlamp, Lens.

> **This spec EXTENDS the approved Phase 04 contract** (`.planning/milestones/v1.0.0-phases/04-gates-observability-dashboard-cli/04-UI-SPEC.md`), whose tokens are live in `dashboard/web/src/index.css` (`@theme` block, verified 2026-07-04). All spacing, typography, color, and voice rules from 04-UI-SPEC are inherited verbatim. This document declares only (a) what this phase reuses, (b) the deltas it introduces, and (c) locked copy for the new states. Where the two conflict, this spec wins for Phase 37 surfaces only.

> **Locked upstream:** CONTEXT.md D-01..D-15 (git artifact transport, panel reuse, drag-to-resize subset, four-state log drawer, read-only lock). RESEARCH.md resolves R-01..R-05 and Pitfalls 1–11. Do NOT re-derive these.

| Property | Value |

**New npm dependencies this phase (frontend only):**

| Package | Version (pinned) | Purpose | Legitimacy |
|---------|------------------|---------|------------|
| `react-markdown` | 10.1.0 | Markdown-render `*.md` artifacts — XSS-safe vDOM output, no `dangerouslySetInnerHTML` | OK — audited in 37-RESEARCH.md (24.6M dl/wk, remarkjs org, no postinstall) |

**Bundle gate:** inherited — <500KB gzipped total, enforced by `dashboard/web/src/__tests__/bundle-size.test.ts`. Measured headroom ~232KB; react-markdown+remark-gfm ≈ 35–45KB gz. Run `make dashboard-frontend` before committing; `verify-dashboard-freshness` requires the embedded dist rebuilt.

## Spacing Scale

Inherited unchanged from 04-UI-SPEC / `index.css`: 4, 8, 12, 16, 24, 32, 48, 64 (`--spacing-1..16`, 8-point base, multiples of 4). Arbitrary values (`mt-[13px]`) remain forbidden.

**New exceptions this phase (additions to 04-UI-SPEC's three):**

| Exception | Value | Rationale |
|-----------|-------|-----------|
| Resize-handle track (panel + log area) | 6px visible, 12px hit area via `before:` pseudo-element | Matches the existing pane-divider exception exactly — one grip idiom everywhere |
| Collapsed-panel rail | 32px wide | Minimum comfortable click/touch target for the expand chevron; off the 8-scale deliberately (it is a control, not content spacing) |
| Markdown table cell padding | 6px 10px | Rendered-document density inside ArtifactViewer only; 8px/12px reads too airy for GFM tables at 13px text |

No other off-scale values. All panel/card/strip padding uses `space-3`/`space-4`/`space-6` exactly as `TaskDetailDrawer` does today (`px-6 py-4` sections, `gap-2`/`gap-3` rows).

## Typography

Inherited: Body 14px/400/1.5 · Label 12px/600/1.4 · Heading 18px/600/1.3 · Mono 13px/400/1.4. Weights 400 + 600 only; no bold 700+, no italics.

**One scoped addition — the rendered-markdown ladder.** Arbitrary planning documents need a heading hierarchy the app chrome never needed. Inside the ArtifactViewer prose container ONLY (class-scoped, never in chrome):

| Markdown element | Size | Weight | Family | Notes |
|------------------|------|--------|--------|-------|
| `h1` | 18px | 600 | sans | Same as app Heading — an artifact's title is a drawer-title peer |
| `h2` | 16px | 600 | sans | **The one new size this phase.** Scoped to rendered artifacts; forbidden in app chrome |
| `h3`–`h6` | 14px | 600 | sans | Collapse the rest of the ladder — density over hierarchy theater |
| body / `p` / `li` | 14px | 400 | sans | line-height 1.5 |
| `code` (inline) | 13px | 400 | mono | On `--color-surface-overlay`, 2px 4px padding, 4px radius |
| `pre` blocks | 13px | 400 | mono | Same treatment as ErrorState's hint `<pre>`: `--color-surface-overlay` bg, 12px padding, 4px radius, `overflow-x: auto` |
| `table` | 13px | 400 (th 600) | sans | 1px `--color-border-subtle` row separators; th in `--color-text-muted` uppercase 11px letter-spacing 0.04em (matches MetaRow `dt` idiom) |
| `a` | 14px | 400 | sans | `--color-text-primary` + underline; `target="_blank" rel="noreferrer"`; the ONE underline surface (04-UI-SPEC anchor rule) |
| JSON pretty-print (`children/*.json`) | 13px | 400 | mono | `JSON.stringify(x, null, 2)` in the same `pre` treatment; no syntax-highlight library (bundle) |

Everything else in this phase (settings cards, approve strip, state placeholders, tab strip) uses the existing four roles only.

| Role | Token | Phase 37 usage |
|------|-------|----------------|
| Dominant (60%) | `--color-surface-base` | Page background (unchanged) |
| Secondary (30%) | `--color-surface-raised` | NodeDetailPanel body, settings cards, approve strip background |
| Secondary sub-tier | `--color-surface-overlay` | Code blocks, JSON viewport, log viewport, raw-spec YAML block, hover states, collapsed rail hover |
| Accent (10%) | `--color-accent` | ONLY per the reserved list below |

**Accent reserved-for list — inherited 6 items, plus one clarification, no additions:**

2. Selected-node ring. 3. Active wave-band edge. 4. SSE-connected dot. 5. Focus ring outline. 6. Connected pill background.

Explicitly NOT accent: resize handles (`--color-border-strong` on hover/drag), selected artifact tab (underline in `--color-border-strong` + `--color-text-primary` label), settings-card headers, collapse chevrons, the Reconnect button (secondary variant), markdown links.

| State | Icon color / accent |
|-------|--------------------|

## Component Inventory

Six component contracts. Planner uses each row as a task-brief seed; file paths follow RESEARCH.md's recommended structure.

### 1. `<NodeDetailPanel>` — generalization of `TaskDetailDrawer` (D-05, D-06, D-09)

The existing right-side panel (`fixed top-0 right-0`, focus trap, Escape-close, focus-return, `role="dialog"` — all a11y solved in `TaskDetailDrawer.tsx`) becomes the shell for ALL Planning-DAG node kinds. `NodeClickContext` gains a kind-aware signature `(kind, name)`; `TideNodeShell` sets `clickable` for all Planning kinds (RESEARCH Pattern 5).

- **Content routing:** `project` → `<ProjectSettingsPanel>`; `milestone` | `phase` → `<ArtifactViewer>`; `plan` → existing PlanDetail content + `<ArtifactViewer>` section; `task` → existing task drawer (unchanged).
- **Header:** mono 18px/600 title `<kind>/<name>` (exact existing idiom: `task/{name}`), X close button (existing 32px hit target).
- **Resize (D-06):** drag handle on the panel's LEFT edge. Width default 420px; clamp [360px, 70vw]. Persist to `localStorage['tide.dashboard.panel-width']` on drag-end (debounced), restore on mount — mirrors the split-ratio persistence idiom.
- **Collapse (D-06):** a chevron button in the panel header collapses the panel to a 32px rail pinned to the right edge showing a rotated `PanelRightOpen` icon; click/Enter expands to the persisted width. Persist `localStorage['tide.dashboard.panel-collapsed']`. Collapse ≠ close: Escape/backdrop still CLOSES; collapse keeps node selection alive.
- **Deep links:** keep `#/plan/<name>`; add `#/project/<name>`, `#/milestone/<name>`, `#/phase/<name>` if cheap (RESEARCH Open Q3 — planner scopes).

### 2. `<ArtifactViewer>` (DASH-01, D-03, D-04, D-07, R-04)

Renders one node's planning artifacts fetched from `GET /api/v1/nodes/{kind}/{name}/artifacts`.

- **File tab strip** (top, when >1 file): horizontal row of mono 12px file names, `children/*.json` entries prefixed `children/`. First `*.md` selected by default. Selected tab: `--color-text-primary` + 2px bottom border `--color-border-strong`. Unselected: `--color-text-muted`, hover `--color-surface-overlay`. Keyboard: tabs are buttons in a `role="tablist"`, arrow keys move selection.
- **Content viewport:** `*.md` → react-markdown + remark-gfm with the `components` prop mapped to the Typography ladder above (NO rehype plugins). `children/*.json` → pretty-printed `pre` block. Vertical scroll inside the panel; **no truncation of any kind, ever** (D-03) — a 2MB artifact scrolls.
- **Typed states (the API returns a discriminator, never an empty list — R-04):**

| State | Render |
|-------|--------|
| `available` | tab strip + content |
| `materializing` (node gate-parked, artifacts not yet on remote — R-01 window) | `Loader2` spinner + heading/body per Copywriting Contract; poll endpoint every 10s while panel open AND node gate-parked; stop on `available` |
| `absent` (pre-upgrade run, or planner-not-yet-run node — Pitfall 9) | static copy per contract, no spinner, manual "Check again" secondary button |
| `no-git` (Project has no `spec.git`) | static copy per contract; no retry (nothing will change) |
| `error` (fetch/creds/network failure) | error copy + "Retry" secondary button — NEVER conflated with absent/no-git |

- **Semantic lock (Pitfall 9):** the panel for node X shows the artifacts **X's own planner produced** (X's planning `*.md` + `children/*.json` describing X's children) — never the parent artifact that specified X.

### 3. `<ApproveStrip>` (DASH-01 gate-parked, D-08)

Pinned strip at the BOTTOM of the panel (sticky within the panel scroll container — artifact content scrolls above it, D-08 "artifact renders above it") shown only when the node's status is `AwaitingApproval`.

- **Layout:** `--color-surface-raised` background, `border-t 1px --color-border-subtle`, `px-6 py-4`. Row 1: existing `<StatusBadge status="AwaitingApproval">` + label-size muted text. Row 2: two `<ClipboardCopyAction>` buttons (existing component, existing toast contract, verbatim):

| Button | Variant | Command copied |
|--------|---------|----------------|
| `Approve` | primary | `tide approve <project>` |
| `Reject` | destructive | `tide reject <project>` |

- Read-only lock preserved: clipboard-copy only, no mutation surface, no confirmation modal (running the pasted command IS the confirmation — inherited D-D6 rationale).

### 4. `<ProjectSettingsPanel>` (DASH-03, D-10, D-11)

Rendered inside `<NodeDetailPanel>` on ProjectNode click. Top-to-bottom:

1. **Live-status strip** (compact, `--color-surface-raised`, `py-3`): the Project CR lifecycle state as a `<StatusBadge>`-style mono chip **labeled `Status`, never `Phase`** (D-11 vocabulary lock — avoids collision with TIDE Phase CRs); budget as mono text `spent $X.XX of $Y.YY cap`; blocking conditions (if any) as the existing ConditionBadge idiom.
2. **Settings cards** — each card: `--color-surface-raised` bg, 1px `--color-border-subtle` border, 4px radius, `p-4`, card header in the MetaRow `dt` idiom (11px uppercase muted, letter-spacing 0.04em). Card order and contents (server redacts — secret NAMES only, values never leave the backend):

| Card | Fields |
|------|--------|
| Outcome prompt | Full `spec.outcomePrompt` in a mono 13px `pre-wrap` block on `--color-surface-overlay` — pre-block, NOT markdown (operator input rendered verbatim; discretion decided, see Assumptions) |
| Repository | Target repo URL (mono), `baseRef` (mono; shows `HEAD (default)` when absent), run branch name if stamped |
| Models | Per-level model IDs (mono), one row per level. **Models only — no effort column** (Pitfall 11: no Effort field exists in v1.0.7); render a muted note `effort: not yet configurable` |
| Budget | Cap, spent, remaining (mono) |
| Gates | Approval policy per level (plain text) |
| Secrets | Secret ref NAMES only (mono), each suffixed muted `(name only — value not shown)` |
| Raw spec | Collapsible (`<details>`-style disclosure, chevron + label per Copywriting Contract); expanded → server-rendered YAML in the standard `pre` treatment |

3. No new nav, no third ViewSwitcher tab (D-09).

### 5. `<PodLogStreamer>` — four-state extension (DASH-04, D-12..D-14)

Existing component extended; existing toolbar/ANSI/auto-scroll contracts unchanged. The state machine gains named-SSE-event mapping (RESEARCH Pattern 4) and **every state renders explicit copy — no state may render an empty viewport**:

| State | Trigger | Render | Reconnect behavior |
|-------|---------|--------|--------------------|
| `connecting` | initial open | `Loader2` + locked copy (existing) | — |
| `connected`, 0 lines | stream open, nothing yet | locked copy (existing) | — |
| streaming | lines arriving | log lines (existing) | — |
| `reconnecting` | transport `onerror`, retry pending | **NEW placeholder** — spinner + locked copy (the missing state that caused the empty drawer) | automatic backoff continues |
| `pod-gone` | named SSE event `pod-gone` | muted message ONLY (D-13) — no pointer, no retry button | **reconnect suppressed** — clear timer, never reopen |
| `stream-error` | named SSE event `error`, or repeated transport failures | `AlertTriangle` in `--color-destructive` + locked copy + `Reconnect` secondary button (D-14) | manual only — button resets backoff and reopens |
| `idle-closed` | named SSE event `idle-timeout` | existing copy + `[Reconnect]` (unchanged) | manual |

Backend widening (invisible but contract-relevant): `resolvePodName` serves Succeeded/Failed-but-present pods, so a finished pod streams retained logs then ends with an honest `pod-gone` frame.

### 6. `<ResizeHandle>` (D-06 — shared, hand-rolled)

One primitive, two instances: panel-width (vertical track, left edge of NodeDetailPanel) and log-area-height (horizontal track, top edge of the log region).

- **Visual:** 6px track in `transparent`; on hover/drag/focus the track shows `--color-border-strong`. 12px hit area. Cursor `col-resize` / `row-resize`.
- **A11y:** `role="separator"`, `aria-orientation`, `aria-valuenow/min/max` (percentage); arrow keys adjust ±16px per press; Home/End jump to clamp limits — mirrors the existing pane-divider keyboard contract.
- **Persistence:** `localStorage['tide.dashboard.panel-width']`, `['tide.dashboard.log-height']`, written on drag-end/key-release (debounced).
- **Clamps:** panel [360px, 70vw]; log area [120px, 70% of panel height].
- **Motion:** no animation during drag (direct style write); collapse/expand uses the existing 180ms `cubic-bezier(0.4,0,0.2,1)` slide; `prefers-reduced-motion` honored globally by `index.css`.

## Out-of-Scope (Explicit Deferrals)

The planner MUST NOT plan tasks for:

- **IDE-style dockable layout for all views** — deferred own phase (CONTEXT `<deferred>`). This phase ships ONLY panel-width + log-height resize/collapse.
- Log archiving / post-GC log persistence — pod-gone renders a message only (D-13; REQUIREMENTS Out of Scope).
- In-browser approve/reject mutation — read-only lock unchanged.
- Syntax highlighting for JSON/YAML blocks — plain mono pre; bundle discipline.
- Search/filter inside rendered artifacts — browser-native `Cmd-F`.
- A "telemetry disabled" banner — that is Phase 38 (TELEM-03), not this phase.
- Any truncation/size-cap UI — D-03 forbids it end-to-end; size failures surface as the typed `error` state.

## Assumptions (headless run — no user questions asked)

Recorded per orchestrator directive; all resolve Claude's-Discretion items conservatively toward CONTEXT.md locks:

1. **Outcome prompt renders as a mono `pre-wrap` block, not markdown** (D-10 discretion). Prompts are operator input, not authored docs — verbatim rendering is the honest display; markdown-rendering could silently reflow meaningful whitespace.
2. **Markdown h2 = 16px** is the single typography addition, scoped to the ArtifactViewer prose container. Chrome keeps the locked 3-size system.
3. **Collapse target is a 32px rail** (not full-hide) so the reopen affordance stays visible — "collapsed like VS Code panels" per the user's framing.
4. **Approve strip commands** use the project-level `tide approve <project>` / `tide reject <project>` forms — the syntax verified in RESEARCH and already locked in 04-UI-SPEC §10 row 1. If Phase-34/36 landings change gate-target CLI syntax, the executor updates the command template, not the button copy.
5. **Poll interval 10s** for the materializing state, only while panel open + node gate-parked + state absent/materializing (RESEARCH Open Q2 recommendation adopted).
6. **Selected-tab affordance uses `--color-border-strong`**, keeping accent reserved to the inherited 6-item list.
