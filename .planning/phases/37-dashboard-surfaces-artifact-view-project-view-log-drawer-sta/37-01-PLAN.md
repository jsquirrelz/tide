<objective>
Fix DASH-04: the silently-empty log drawer. Root cause is verified in code (RESEARCH Pitfall 1, three stacked causes): (1) the backend's named terminal SSE events (`pod-gone`, `error`, `idle-timeout`) are never listened for — `sse.ts` registers only unnamed `message` frames and project-event names; (2) the `reconnecting` state renders no placeholder; (3) the server closes the stream after terminal frames, so EventSource auto-reconnects forever against a GC'd pod. Plus one backend gap: `resolvePodName` only returns Running/Pending pods, so a finished-but-present pod is misreported as pod-gone.

Implements D-12 (four-state model: loading → streaming → pod-gone | error), D-13 (pod-gone = honest message only, no retry), D-14 (error state distinct from pod-gone, carries a manual reconnect affordance).

Purpose: DASH-04 — the log drawer always renders an explicit state, never silently empty.
Output: named-event support in sse.ts, terminal state machine in useTaskLog, complete placeholder coverage in PodLogStreamer, widened resolvePodName.
</objective>
