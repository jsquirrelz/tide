<objective>
Expose the two read-only endpoints the Phase-37 UI consumes: node artifacts (DASH-01) and project settings (DASH-03), plus the RBAC expansion that lets the dashboard read git-creds Secrets at fetch time (R-02). Both handlers follow the plans.go/projects.go idiom exactly — GET-only chi routes, writeJSON/writeError helpers, empty-array-not-null serialization, whitelist response structs.

Security posture is the point: settings redaction happens SERVER-side (secret names only — values never leave the backend); Secret reads use the typed clientset (a cached-client Get would silently start a cluster-wide Secret informer — Pitfall 4); the RBAC delta is exactly `secrets: get`, which keeps `make helm-rbac-assert`'s ⊆{get,list,watch} invariant.

Purpose: DASH-01 + DASH-03 backend — the panel surfaces have something to fetch.
Output: artifacts + settings handlers with tests, router registration, main.go wiring, RBAC delta.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → dashboard API | Untrusted URL params (kind/name/project/namespace) |
| dashboard → K8s Secrets | New read capability on git-creds Secrets |
| dashboard → git remote | Authenticated fetch with per-project PAT |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-37-07-01 | Information Disclosure | secret values via settings response | critical | mitigate | Names-only whitelist; no Secret read in the settings handler at all; raw-body sentinel test proves zero value bytes in responses (Task 2 Test 2) |
| T-37-07-02 | Elevation | dashboard SA privilege creep | high | mitigate | Exactly `secrets: get` (no list/watch); make helm-rbac-assert gate; typed clientset only (no informer); chart comment documents scope + rationale |
| T-37-07-03 | Information Disclosure | PAT leakage via error states/logs | high | mitigate | Error messages tested to exclude the PAT value; credential lives only in the fetch call frame (37-03 contract); never logged |
| T-37-07-04 | Tampering | path traversal via kind/name params | medium | mitigate | Kind allowlist (400 otherwise); name/namespace go into typed K8s Gets and a prefix filter over a git-tree walk — never into filesystem or shell path concatenation |
| T-37-07-05 | Denial of Service | repeated large fetches | medium | mitigate | Tip-check + bounded LRU (37-03); request ctx cancellation; D-03 forbids trimming so the guard is cache eviction, never content truncation |
| T-37-SC | Tampering | npm/pip/cargo installs | low | accept | No package installs in this plan |
