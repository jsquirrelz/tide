<objective>
Build the artifact-viewing content components (D-07, D-08) and the typed fetch layer for both new endpoints. ArtifactViewer renders one node's planning artifacts with a file tab strip, markdown/JSON rendering, and the full R-04 typed-state contract; ApproveStrip is the D-08 gate-parked pinned strip. Both are standalone components mounted into NodeDetailPanel by plan 37-08 — no App wiring here.

The API contract is fixed by this plan set (37-07 implements the Go side to the same shape): the server returns a state discriminator `available | absent | no-git | error`; the UI derives `materializing` = state absent AND node gate-parked (R-01's eventual-consistency window), polling every 10s while the panel is open in that state (Open Q2 recommendation adopted).

Purpose: DASH-01 — operators read the artifacts a node produced, and approve decisions need no PVC reader pod.
Output: react-markdown@10.1.0 + remark-gfm@4.0.1 installed (audited OK); ArtifactViewer, ApproveStrip, api.ts fetchers + types.
</objective>
