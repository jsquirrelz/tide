<objective>
Gap 37-G1 closure. The dashboard artifact view returns `state:"error"` (`git credentials secret "…" is missing data key GIT_PAT`) when a Project's `git.credsSecretRef` points at a Secret with an empty/absent `GIT_PAT` AND the `repoURL` is an anonymous `http://` remote — even though the run branch has the artifacts. Live-repro'd during the 37-10 UAT: an in-cluster anonymous http remote pushed fine, but the artifact panel stayed `state:error` until a dummy PAT was set.

Root cause: `resolveAuth` in `cmd/dashboard/api/artifacts.go` treats a set `credsSecretRef` with a missing/empty `GIT_PAT` as a hard error. It lacks the scheme-conditional relaxation `cmd/tide-push/main.go` `resolveGitAuth` already applies — empty PAT ACCEPTED for `http://`, REQUIRED for `https://`/`git@`.

Fix: pass the repo URL into `resolveAuth` and mirror the push path's `requirePAT` guard — for a non-authenticated scheme (`http://`) an empty/absent `GIT_PAT` proceeds anonymously (nil Auth); for `https://`/`git@` the missing-key error is preserved.

Purpose: DASH-01 — anonymous in-cluster http remotes render their artifacts without a dummy PAT.
Output: scheme-conditional `resolveAuth` + two regression tests (http://→available, https://→error), no other behavior change.
</objective>
