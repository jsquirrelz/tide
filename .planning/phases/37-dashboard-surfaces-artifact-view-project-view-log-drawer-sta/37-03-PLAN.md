<objective>
Build the dashboard's git read path (R-02's mechanism, D-01's "manager fetches from the remote, caches, and streams"): a new `cmd/dashboard/gitfetch` package with (1) a Fetcher interface seam and a go-git shallow-clone implementation that reads the `.tide/` tree at a branch tip, and (2) a bounded LRU Store over it. K8s-free by design — Secret resolution stays in the API handler (plan 37-07), so this package is pure git + caching and fully testable against local fixture repos.

go-git sharp edges are designed around, not patched around (RESEARCH Pattern 3): fresh clone per new tip SHA (never pull into a shallow clone), Tags:NoTags, SingleBranch, NoCheckout, in-memory storage discarded after tree extraction.

Purpose: DASH-01 — the artifact viewer needs artifacts fetchable from the remote; this is the only cross-namespace-safe read path (manager cannot mount project PVCs).
Output: gitfetch package with interface seam, shallow fetcher, LRU store; golang-lru promoted to direct dependency.
</objective>
