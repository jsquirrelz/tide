<objective>
Fix the budget-bypass resume path so a halted, already-initialized Project resumes at `Running`
(not `Pending`) and never re-dispatches the init or clone Jobs against an already-initialized
workspace. The clone guard uses the durable `CloneComplete` flag (added in Plan 01), not
TTL-unreliable Job existence.

Purpose: BYPASS-01 (`project_controller.go:1257` sets `PhasePending`, which re-enters init-Job
dispatch and wipes the workspace) and BYPASS-02 (clone dispatch guarded only by `IsNotFound`, which
is true after the 300s clone-Job TTL expires) are the two correctness bugs that make a budget resume
destructive.
Output: Corrected bypass target phase, an init-Job BranchName guard, a `CloneComplete`-gated clone
dispatch with set-on-success, and two envtests.
</objective>
