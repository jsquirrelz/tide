## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runBranch name → tide-push CLI | CloneOptions.RunBranch flows from Project.Status.Git.BranchName (controller-set) into --run-branch arg |
| integrateBranches → tide-push CLI | Task UIDs come from K8s API; TaskBranchName(uid) is deterministic; CSV passed as --integrate-task-branches |
| git worktree add path | runWorktreeDir is a computed path under cfg.Workspace; not user-controlled |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-03-01 | Injection | --run-branch value passed to exec.Command("git", ..., "worktree", "add", ..., runBranch) | mitigate | RunBranch value originates in ProjectReconciler (tide/run-<project.Name>-<unix>): project.Name is a K8s name (DNS subdomain, validated by apiserver) and unix timestamp. exec.Command does not invoke a shell; no shell injection path. |
| T-11-03-02 | Injection | --integrate-task-branches CSV passed to tide-push via job arg | mitigate | Branch names are TaskBranchName(task.UID) where UID is a K8s UUID. The CSV is split by splitCSV() in tide-push and each element is passed as a positional git merge arg. No shell invoked. |
| T-11-03-03 | Tampering | --force-with-lease lease correctness (existing push uses LastPushedSHA) | accept | The integration step runs BEFORE the planner-artifact commit in runPush; the lease is checked on the final push (unchanged from existing behavior). IntegrateTaskBranches modifies the local run branch; the lease anchor (LastPushedSHA) correctly reflects the last PUSHED SHA, which still matches the remote state at push time. |
| T-11-03-04 | Elevation of Privilege | tide-push clone job passes --run-branch; could provision a branch for a different project | mitigate | RunBranch is project.Status.Git.BranchName (controller-controlled, format-validated as tide/run-<name>-<unix>). The clone job is owned by the Project (owner ref), runs in the project namespace with the project PVC. Cross-project collision is structurally prevented by the deterministic branch name scoped to project.Name. |
</threat_model>
