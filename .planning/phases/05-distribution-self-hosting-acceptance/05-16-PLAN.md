<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI workflow → ghcr.io OCI registry | `helm push` requires `packages: write` token scope; misconfigured permissions either fail the push silently (operator surprise) or over-grant the token (security risk) |
| Tag-triggered workflow → goreleaser binary release | release.yaml's `pre-flight` job blocks non-rc releases if no successful rc dry-run; without this gate, regression-bearing releases could ship |
| DinD socket mount in dry-run.yaml runner | The runner mounts /var/run/docker.sock; the dry-run script gets full Docker daemon access — equivalent to root on the runner; this is the canonical DinD pattern but worth noting in threat model |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-16-01 | Elevation of Privilege | chart-publish job permissions | mitigate | `permissions: contents: read + packages: write` — minimum needed for `helm push`; explicit per Pitfall 3; helmify-verify job has `contents: read` only (no write) |
| T-05-16-02 | Information Disclosure | GITHUB_TOKEN in helm registry login | mitigate | `--password-stdin` (never on command line); login → push → logout sequence with `if: always()` logout safety net |
| T-05-16-03 | Tampering | Chart tree drift on release | mitigate | helmify-verify job runs `make helm && git diff --exit-code charts/`; fails release if drift detected (Pitfall 1 mitigation) |
| T-05-16-04 | Tampering | Releasing v1.0.0 without prior rc dry-run | mitigate | pre-flight job uses `gh run list --workflow dry-run.yaml` to require a successful rc tag dry-run before any non-rc release; fails the workflow if missing |
| T-05-16-05 | Elevation of Privilege | DinD socket access in dry-run runner | accept | Canonical DinD pattern with --network host + bind-mounted /var/run/docker.sock (Pitfall 6 lock); no --privileged; runner is ephemeral ubuntu-latest with no persistent state |
| T-05-16-06 | Information Disclosure | Dry-run-report.json artifact retention | accept | retention-days 90 (GitHub default); contains tool versions + per-phase elapsed seconds, no secret material; safe to retain at default duration |
</threat_model>
