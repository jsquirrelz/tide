## Gate Decision: BLOCKED

`gate_decision: BLOCKED` ŌĆö cascade-8 is confirmed CLOSED (`dispatch_active: true`; Jobs created for alpha and beta Tasks in `tide-int-test` namespace) but cascade-9 surfaced three sub-classes:

**Sub-class A: job.spec.activeDeadlineSeconds=60 too tight for credproxy initContainer under kind**
- Root cause: `PodJobBackend.buildJob()` hardcodes `activeDeadlineSeconds: 60`. Under kind, the credproxy initContainer (`envelope-writer` + `tide-credproxy`) takes >60s to start. The Job transitions to `DeadlineExceeded` before the main container runs.
- Evidence: `kubectl describe job -n tide-int-test` confirms `DeadlineExceeded`; AC1's `Eventually(3m)` for `task.Status.Phase = Succeeded` never fires.

**Sub-class B: Layer A envtest SUB-02 spec-flake**
- Root cause: `TaskReconciler` now runs `reconcileDispatch` when `Dispatcher != nil`. In the envtest full-suite context, tasks in the indegree test namespace lack a parent Project, so `resolveProject` returns error ŌåÆ reconcile requeue loop. This requeue load competes with the "Wave roll-up: Succeeded when all Tasks Succeeded" spec's 20s window, causing the Wave's `taskToWaveMapper` watch to be delayed.
- Evidence: The spec passes (2/2) when run in isolation with `-label=SUB-02`; fails in the full-suite context.

**Sub-class C: Wall-time budget exceeded**
- `clean_run_wall_seconds: 1486` >> 600s budget. BeforeSuite=127s (helm install + wait) + AC1 3m Eventually + overhead exceeds the Makefile `timeout 600s`.
