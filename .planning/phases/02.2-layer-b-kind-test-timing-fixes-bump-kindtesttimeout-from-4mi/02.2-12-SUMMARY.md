## What This Plan Did

### Task 1 — Source Edits

**Cascade-10 closure — architectural pivot (vs Plan 02.2-11-SUMMARY's Option A):**

The prior cascade-10 fix-shape proposed in Plan 02.2-11-SUMMARY was Option A: "stamp `tide-projects` PVC declarations into every test fixture YAML." This plan rejected Option A in favor of an architecturally cleaner approach:

1. **`internal/dispatch/podjob/backend.go` — new `PodStatusEnvelopeReader`:** reads `EnvelopeOut` from `Pod.status.containerStatuses[].state.terminated.message` using a label-selector query `client.MatchingLabels{"tideproject.k8s/task-uid": taskUID}`. Manager no longer needs PVC visibility for envelope reading.
2. **`EnvelopeReader.ReadOut` signature change:** `(ctx, taskUID)` → `(ctx, projectUID, taskUID)`. All implementations and test mocks updated (`internal/controller/suite_test.go`, `test/integration/envtest/suite_test.go`).
3. **`FilesystemEnvelopeReader` path update:** workspace path now `WorkspaceRoot/{projectUID}/workspace/envelopes/{taskUID}/out.json` (matches prod layout; was previously flat `WorkspaceRoot/envelopes/{taskUID}/out.json`). The Filesystem reader stays as the fallback for in-process / local-test paths.
4. **`cmd/manager/main.go`:** wires composite `PodStatusEnvelopeReader{Client: mgr.GetAPIReader(), Fallback: &FilesystemEnvelopeReader{WorkspaceRoot: "/workspaces"}}`.
5. **`cmd/stub-subagent/main.go`:** new `writeTerminationMessage(data)` helper writes envelope JSON to `/dev/termination-log` (overridable via `TIDE_TERMINATION_MESSAGE_PATH` for tests).
6. **`test/integration/kind/suite_test.go` — `ensureProjectsPVC(ns)`:** retains a namespace-local PVC for Pod-side mount (RWO 1Gi); called from `createNamespace(ns)` alongside `ensureSubagentSA(ns)`.
7. **`test/integration/kind/testdata/three-task-wave.yaml`:** inline PVC declaration so `kubectl apply -f` is self-contained.

**Cascade-11 closure — direct namespace-local Secret mirroring:**

- **`ensureSigningKeySecret(ns)` + `controllerSigningKeyData()` + `signingKeySecretYAML(ns, keyData)`** in `test/integration/kind/suite_test.go`. Mirrors `tide-system/tide-signing-key` into every test namespace using a `kubectl get secret -o jsonpath` read followed by a base64-data-identical apply. ## Pattern observation (per CLAUDE.md Operating Note #4)

The cascade chain shows distinct classification phases:

- **Iterations 1–4** — predictable tactical (chart contract + flag-mismatch + budget): each closure plan ≤ 1-line edits.
- **Iterations 5–7** — harness-bug class (fixture incomplete): credproxy fixture, then caps/output/failure fixtures. All `applyHierarchy` + `createProjectHierarchy` ensure-helper pattern.
- **Iteration 8** — production-wiring-gap (Dispatcher field nil): the STRUCTURAL blocker that all prior cascades were SYMPTOMS of. Single-line wiring fix in `cmd/manager/main.go`.
- **Iteration 9** — multi-class (A=Pod-deadline + B=spec-flake + C=wall-budget): three independent surfaces from the correct dispatch wiring.
- **Iteration 10/11 (cascade-10 + cascade-11)** — harness-bug class (namespace-local chart resources): PVC + Secret. Closed together via the ensure-helper pattern + an architectural pivot in cascade-10's case.

The pattern: each cascade reveals progressively-deeper layering assumptions. Cascade-8 was the keystone — closing it forced cascade-9/10/11 to surface in sequence as the runtime now actually executed the dispatch path. After cascade-11, the harness-bug class is exhausted (every namespace-local chart resource — SA + PVC + Secret — is now mirrored).
