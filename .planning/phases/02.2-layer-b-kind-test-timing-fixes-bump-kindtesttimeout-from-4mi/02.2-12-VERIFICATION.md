- `pvc_not_found_event_count: 0` in frontmatter
- Cascade-10 sub-classifies as **CLOSED** via the architectural pivot:
  - Manager-side: `EnvelopeReader` is now `PodStatusEnvelopeReader` which reads `EnvelopeOut` from the subagent container's terminationMessagePath via `Pod.status.containerStatuses[].state.terminated.message`. The manager no longer requires PVC visibility.
  - Pod-side: the PVC remains namespace-local (a fixture concern). Why this architectural pivot (vs Plan 02.2-11-SUMMARY's proposed Option A "stamp PVC into every fixture"): the manager mounting a namespace-local PVC was always a layering smell — the controller process and the workload Pods run in different namespaces, and PVC RWO semantics on kind's local-path provisioner can't bridge them. The Pod-status transport removes the cross-namespace assumption entirely; the PVC remains a Pod-local concern that test harnesses provision per namespace.
