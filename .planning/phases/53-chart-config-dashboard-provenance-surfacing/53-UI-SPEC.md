## Component Contract 1 Рђћ TaskDetailDrawer "Verification" section

**Placement:** new section in `TaskDetailDrawer.tsx` immediately AFTER the metadata grid and
BEFORE the `PhoenixTraceLink` mount. Section anatomy: `border-t border-[var(--color-border-subtle)]
px-6 py-4` (the existing drawer section-separator convention).

**Eligibility (owned by the section, PhoenixTraceLink precedent Рђћ mount point never
re-decides):** renders ONLY when the task payload carries a verification contract (new
payload fields present/non-null). A task with no verification contract shows no section at
all Рђћ absence renders nothing, zero noise.

**Internal layout, top to bottom:**

1. **Section header** Рђћ text `Verification`, MetaRow `<dt>` convention: 11px, uppercase,
   `letterSpacing: 0.04em`, `--color-text-muted`.
2. **2-column `<dl>` grid** (`grid-cols-2 gap-x-4 gap-y-3`, values 13px), rows in order:

   | Label (dt) | Value (dd) | Format |
   |------------|-----------|--------|
   | `Iteration` | `{LoopStatus.Iteration} of {spec.verification.maxIterations}` | plain text (matches existing "Attempt" row format) |
   | `Verdict` | `LastEvaluation.Decision` | mono, colored: `APPROVED` Рєњ `var(--color-status-success)`, `REPAIRABLE` Рєњ `var(--color-status-warning)`, `BLOCKED` Рєњ `var(--color-status-blocked)`; when `LastEvaluation` is nil Рєњ `No verdict yet` in `--color-text-muted` (not colored, not mono) |
   | `Findings` | `{FindingsCount} total ┬и {HighSeverityCount} high` | plain; `Рђћ` when no evaluation yet |
   | `Exit reason` | `LoopStatus.ExitReason` | mono; `Рђћ` when empty (loop still open) |
   | `Loop run` | derived `loopRunID` | mono, `truncate`, `title` tooltip = full ID (Envelope-path row precedent) |
   | `Attempt ID` | derived `attemptID` | mono, `truncate`, `title` tooltip = full ID |

3. **Findings link row** Рђћ see Component Contract 2.

**The attemptMax/maxIterations firewall (Phase-51 D-05 distinction, non-negotiable):**

- The EXISTING metadata-grid row `Attempt Рђћ {attempt} of {attemptMax}` keeps deriving from
  `Caps.Iterations` (infra-retry cap). It is NOT moved, NOT relabeled, NOT re-sourced.
- The NEW `Iteration Рђћ N of M` row derives from `LoopStatus.Iteration` /
  `spec.verification.maxIterations` (quality-iteration bound).
- Two rows, two labels (`Attempt` vs `Iteration`), two data sources. They MUST never be
  merged, cross-wired, or displayed as one number. A drawer test asserts both rows render
  simultaneously with independent values (e.g. Attempt `1 of 3` alongside Iteration `2 of 2`).

**Phoenix deep-link:** the existing `PhoenixTraceLink` mount (directly below this section) IS
the span-level provenance surface Рђћ Project run Рєњ level Рєњ attempt Рєњ tool spans live in
Phoenix, reached via the existing link. No second mount, no change to `PhoenixTraceLink`,
no in-dashboard span rendering. The dashboard shows the current-iteration summary and links
out for history (LOOP-03: no iteration-history timeline exists in CRD status, by design Рђћ
do NOT build one).

## Component Contract 2 Рђћ Findings browsing (existing ArtifactViewer, existing endpoint)

**Trigger:** link labeled `View findings` at the bottom of the Verification section.
Anchor anatomy mirrors `PhoenixTraceLink`'s (the locked link style): mono 13px, underline,
`--color-text-primary`, `hover:bg-[var(--color-surface-overlay)]`, with a leading
`ScanSearch` 14px icon (`aria-hidden`). NOT accent, NOT a button.

**Interaction:** in-drawer disclosure Рђћ clicking toggles an inline area below the link that
renders the task's staged `findings.json` from the EXISTING
`GET /api/v1/nodes/task/{name}/artifacts` gitfetch endpoint (`fetchNodeArtifacts("task", Рђд)`
Рђћ `ArtifactViewer`'s `kind` prop is already an open `string`, Finding 14). The trigger
carries `aria-expanded` + `aria-controls`. No navigation away, no new route, no new endpoint.
Reusing `<ArtifactViewer kind="task" Рђд/>` wholesale inside the disclosure is the sanctioned
implementation; a lighter single-file mono `<pre>` render reusing ArtifactViewer's fetch +
StatePanel states is equally acceptable Рђћ executor's choice, states contract below binds
either way.

**States (reuse ArtifactViewer's StatePanel vocabulary):**

| State | Copy |
|-------|------|
| available | findings.json content rendered (mono) |
| absent (task kind Рђћ new copy) | heading `No findings staged yet` ┬и body `Findings land on the run branch after a verifier attempt completes. Check again after the current iteration finishes.` ┬и action `Check again` |
| error | existing locked copy verbatim: `Couldn't fetch artifacts` / `The dashboard could not read the run branch. Check the dashboard logs: kubectl logs deploy/tide-dashboard` ┬и action `Retry` |
| no-git | existing locked copy verbatim (unchanged) |
