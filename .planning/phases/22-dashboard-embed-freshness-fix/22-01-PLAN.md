<objective>
Close FIX-01: make the dashboard image build regenerate the embedded SPA from source so a published image can never ship a bundle older than `dashboard/web/src`, and add a staleness gate that catches a forgotten `dist/` regeneration in PR CI and at release time.

Purpose: dogfood run #2 found v1.0.0/v1.0.1 dashboard images froze the embedded bundle at a pre-telemetry commit because `Dockerfile.dashboard` only runs `go build` (which `//go:embed`s the hand-committed `cmd/dashboard/embed/dist`) and `release.yaml` never runs `make dashboard-frontend` first. This plan makes freshness automatic (self-contained image build) and enforced (CI gate), without gitignoring `dist/` (Option A — keeping it tracked is required so `go vet`/`make test` compile from a clean clone).

Output: a multi-stage `Dockerfile.dashboard` with a `node:22-alpine` `spa-builder` stage; `.dockerignore` re-includes for the SPA source; a `make verify-dashboard-freshness` target that rebuilds, git-diffs `cmd/dashboard/embed/dist/`, and asserts the `panel-cache-efficiency` telemetry marker; and the gate wired into `ci.yaml` (PR) and `release.yaml` (helmify-verify).
</objective>
