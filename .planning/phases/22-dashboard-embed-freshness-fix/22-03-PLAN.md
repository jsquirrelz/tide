<objective>
Harden the `make verify-dashboard-freshness` gate (the Phase 22 deliverable) against the two robustness gaps the 22-REVIEW.md code review found, so the gate is airtight before later phases build on it.

Purpose: the current gate calls `$(MAKE) dashboard-frontend`, which does `rm -rf cmd/dashboard/embed/dist && cp -r dashboard/web/dist cmd/dashboard/embed/dist` and then `git diff --quiet cmd/dashboard/embed/dist/`. Two consequences:
- WR-01 (dirty-tree-on-failure): on a stale tree the recipe mutates the tracked embed dir in place, then exits 1 — leaving the developer's working tree dirty with a deleted tracked asset plus a new untracked asset that `git checkout` won't clean. Local footgun.
- WR-02 (untracked-asset false-pass): `git diff --quiet` is blind to net-new untracked files. A future build that ADDS a new chunk without modifying any tracked file would not be caught.

Output: a `verify-dashboard-freshness` target that builds the SPA into vite's own output dir (`dashboard/web/dist`) and compares it to the committed `cmd/dashboard/embed/dist` with `diff -rq` — a full recursive tree comparison that catches added, removed, AND changed files, and never writes to the tracked embed dir. This single change closes both WR-01 and WR-02.
</objective>

<task number="1" name="Rewrite verify-dashboard-freshness to diff against a fresh build without mutating the tracked tree">
Rewrite the `verify-dashboard-freshness` recipe in `Makefile` (currently ~lines 283-298) to:

1. Build the SPA into `dashboard/web/dist` directly (not via `$(MAKE) dashboard-frontend`):
   `cd dashboard/web && npm ci && npm run build && npm run test`
2. Compare the fresh build to the committed embed tree without mutating it:
   ```
   @if ! diff -rq dashboard/web/dist cmd/dashboard/embed/dist; then \
       echo "FAIL: cmd/dashboard/embed/dist/ diverges from a fresh SPA build — run 'make dashboard-frontend' and commit the result before merging"; \
       exit 1; \
   fi
   @echo "PASS: cmd/dashboard/embed/dist/ matches a fresh SPA build (added/removed/changed files all checked)"
   ```
