<objective>
Wire the `make verify-dashboard-freshness` gate (created in Plan 22-01) into both CI workflows so a stale embedded `dist/` is caught at PR time and again at release time before any image publishes.

Purpose: ROADMAP success criterion #2 — "CI fails a build whose embedded dist is older than the dashboard source." The Makefile target is the mechanism; this plan is the enforcement wiring. The PR gate (ci.yaml) gives fast feedback; the release gate (release.yaml helmify-verify) is defense-in-depth — the last check before binaries/images ship, mirroring the existing chart-reproducibility gate in the same job.

Output: an `actions/setup-node@v4` + `make verify-dashboard-freshness` step pair in the ci.yaml gates job, and the same step pair appended inside the release.yaml helmify-verify job.
</objective>
