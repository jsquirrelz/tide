<objective>
Generalize `cmd/tide-push/main.go`'s `stageEnvelopeArtifacts` so it can stage a Task-shaped findings artifact (a single `findings.json`, no `*.md`, no `children/`) without the existing planner-`*.md` hard-fail — while keeping the Milestone/Phase/Plan/Project staging path byte-identical.

Purpose: Close the highest-risk plumbing trap in this phase (RESEARCH Pitfall 1). The full-findings tier of the size×locality rule (EVAL-05c) stages findings onto the run branch via the extended `collectStageEnvelopes`/`stageEnvelopeArtifacts` path. Today `stageEnvelopeArtifacts` hard-fails (`artifact-stage-failed`) on any envelope dir with zero `*.md` files. A Task/verifier findings dir has only `in.json`/`out.json`/`findings.json` — no `*.md`. Generalizing the glob NOW prevents Phase 51 from shipping a push-breaking regression the first time any Task's verify iteration completes.

Output: modified `cmd/tide-push/main.go` (`stageEnvelopeArtifacts` + possibly `EnvelopeStage`), new regression case in `cmd/tide-push/main_test.go`.

Scope discipline (LOCKED): Do NOT add a Task entry to `collectStageEnvelopes` (`internal/controller/artifact_push.go`) — nothing produces `findings.json` yet; that is Phase 51's job. This plan only makes the `tide-push` CONSUMER side capable of handling a `task/<name>` entry when one eventually arrives. The change is purely additive and backward-compatible.
</objective>
