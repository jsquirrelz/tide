<objective>
Wire the controller half of the loop-identity + terminal-reason contract: `buildEnvelopeIn` stamps `LoopRunID`/`AttemptID` from existing identity (D-01 — the `podjob.JobName` tuple, derived not minted), the `EnvelopeReadFailed` branch synthesizes `TerminalReason: cap_exceeded` for wall-clock-killed Jobs (the controller half of RESEARCH Open Question 1 — the ONLY place a `DeadlineExceeded` kill can ever be classified, since the SIGKILLed pod never writes out.json), and `synthesizePlannerSpan` stamps the `loop.*` attributes on the Task AGENT span (D-05 primary home).

**Scope fence:** this plan touches reconciler code ONLY at the envelope-synthesis and span-attribute layers. It does NOT change today's exit-0→Complete controller path, adds NO halt class (`grep VerifyHalt` stays 0), does NOT touch `failure_halt.go`, and does NOT alter the `ConditionFailed`/`EnvelopeReadFailed` condition Reason (only its Message gains cap detail). The wave/failure semantics contract is untouched.

Output: dispatch-time identity stamping, cap_exceeded synthesis, and AGENT-span loop attributes — green under envtest.
</objective>
