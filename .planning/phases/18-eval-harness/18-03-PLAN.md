<objective>
Ship the live `count_tokens` pre-flight as a maintainer tool: `cmd/tide-eval/main.go` behind `//go:build eval`, invoked by a new `make eval` target. It renders each of the five templates, POSTs each through the existing credproxy's already-allowlisted `POST /v1/messages/count_tokens` route using stdlib `net/http` (NO Anthropic SDK), and prints per-template real token counts plus a 1,024-token cache-floor pass/fail. This tool produces the authoritative token numbers that Phase 19/20 use to tune ratchets and verify the cache minimum.

Purpose: EVAL-05 (count_tokens via credproxy, stdlib net/http, behind //go:build eval, exposed as `make eval`).
Output: `cmd/tide-eval/main.go` + a `make eval` Makefile target.

This plan is independent of plans 01/02 — no shared files, no test dependency. It is the only online surface in the phase; `//go:build eval` keeps it out of the deterministic `make test` tier (D-02/D-03).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cmd/tide-eval → credproxy | The only network/credential touch in Phase 18. tide-eval holds a signed token and POSTs to the credproxy, which injects the real Anthropic key. The token crosses this boundary in the x-api-key header. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-03-01 | Information Disclosure | signed token in logs/stdout | mitigate | cmd/tide-eval never prints or logs the token value; logs only "token present: true". Token is passed solely to req.Header.Set. Verified in acceptance criteria. |
| T-18-03-02 | Spoofing / fail-open | missing proxy URL or token | mitigate | Both the command (requireFlag → os.Exit(1)) and the Makefile target (guard → exit 1) fail closed when TIDE_PROXY_ENDPOINT or TIDE_SIGNED_TOKEN is absent — no silent fallback to a default endpoint or unauthenticated call. |
| T-18-03-03 | Information Disclosure | rendered template content sent to API | accept | Templates are the project's own committed prompts (no secrets); count_tokens does not generate output. The credproxy is the established egress boundary. |
| T-18-03-SC | Tampering | (no new package installs) | accept | This plan adds no new dependency — stdlib net/http only. No supply-chain surface beyond Plan 01's goldie addition. |
</threat_model>
