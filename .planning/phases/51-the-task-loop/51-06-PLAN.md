<objective>
Build the FORWARD half of the verifier sub-state-machine in `TaskReconciler`: on executor exit-0 (belief-complete, EXEC-04), a Task carrying a verification contract transitions to a `Verifying` sub-state and dispatches an INDEPENDENT verifier Job — gated by `verifierInFlightCount` (cap-before-acquire) + `LoopPolicy.BudgetCents` reservation — with a `VerifyContext`-populated envelope (`Role=verifier`, `Vendor=langgraph`, `GateCommand` from the LOCKED spec, rendered via `task_verifier.tmpl`) and the `lockedSHA` stamped so `git show` reproduces it. A Task with no contract preserves today's exit-0→Succeeded (OQ2 backward-compat).

Purpose: TASK-01 (dispatch the locked spec, record lockedSHA), TASK-04 (verifier is a logically independent process), ESC-04 (concurrency cap + BudgetCents), OBS-03 (SelfInstruments flows langgraph). The BACKWARD half (verdict consumption, fresh attempt, anti-gaming, halt trigger, EVALUATOR span call site) is Plan 07.
Output: verifier dispatch path + verifierInFlightCount + envtest for the dispatch decision + cap ordering.
</objective>

| Boundary | Description |
|----------|-------------|
| executor belief-complete → verifier dispatch | Correctness is NOT stamped from the executor; an independent verifier decides (EXEC-04) |
| manager → verifier gate command | GateCommand flows from the LOCKED, planner-authored contract, never model-supplied |
| dispatch → concurrency/cost budget | Unbounded verifier fan-out (attempts × levels) must be capped before spend |
