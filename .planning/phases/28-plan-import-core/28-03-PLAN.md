<objective>
Build the `cmd/tide-import` in-pod binary (mirroring `cmd/tide-reporter`): it copies
salvaged UID-keyed envelope trees to new-UID paths so the unchanged FilesystemEnvelopeReader
finds them, rewrites `out.json` TaskUID atomically, converts embedded child `Spec.Raw`
bytes to v1alpha2 (strip unknown fields via typed round-trip), strict-validates each child
against the Kind allowlist + completeness check, and refuses any path-traversal escape. Plus
the Dockerfile (distroless, stdlib-only closure) and a table-driven unit-test suite.

This plan owns IMPORT-02 (validate-before-adopt), IMPORT-03 (FQ-name rekey applied to
filesystem paths), and the binary-side of IMPORT-05 (same-workspace containment). The binary
is pure filesystem I/O — no K8s client — keeping it inside the provider firewall (stdlib +
api/v1alpha2 + pkg/dispatch only).

Purpose: The binary is independently testable offline (t.TempDir, no envtest), so it lands
in Wave 2 before the ImportController (Wave 3) that dispatches it as a Job.
Output: binary + Dockerfile + unit tests, all green via `go test ./cmd/tide-import/...`.
</objective>
