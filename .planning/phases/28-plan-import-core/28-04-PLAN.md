<objective>
Build the `ImportController` — the one-shot pre-reconcile state machine
(Pending→CreatingCRs→CopyingEnvelopes→Complete/Failed) that reads the seed ConfigMap,
materializes the Milestone/Phase/Plan CR tree with new UIDs (AlreadyExists = success),
records the FQ-name → oldUID → newUID rekey table, runs `dag.ComputeWaves` on the full task
set BEFORE any `client.Create` (cycle detection — D-10/IMPORT-04, since client.Create
bypasses the admission webhook), dispatches the tide-import Job (two PVC subPath mounts,
this namespace only — IMPORT-05), and on Job success sets `ConditionImportComplete=True`.

This plan owns IMPORT-04 (cycle detection + Wave-never-imported), the controller side of
IMPORT-01 (the condition that clears the 5 guards lands here), IMPORT-03 (rekey table
construction), and the controller/Job side of IMPORT-05 (same-namespace subPath containment
+ Kind allowlist + completeness already enforced by the binary). Idempotency: the first step
checks ImportComplete and returns immediately if True (D-12).

Purpose: This is the structural heart. ProjectReconciler is the exact analog (init-Job state
machine + condition-driven transitions + SetupWithManager). It depends only on the schema
(plan 02); the binary image (plan 03) is referenced by ref string only, so this can run in
the same wave as 03.
Output: controller + jobspec + envtest, green via `go test ./internal/controller/... -run Import`.
</objective>

| seed ConfigMap → client.Create (CRD API channel) | Seed entries materialize into real CRs; client.Create bypasses the validating admission webhook (the central IMPORT-04/05 surface) |
