# Phase 21 — UI Design Contract

> Visual and interaction contract for the cache-efficiency panel + per-level breakdown added to the read-only operator dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This is an ADDITIVE contract on a mature, token-locked dashboard.** It does not introduce a new design system. Every value below is INHERITED from the existing locked tokens in `dashboard/web/src/index.css` (`@theme` block) and the established TelemetryView conventions. New rows describe only the cache-efficiency trio + sparkline panel and the per-level selector. The executor MUST reuse existing tokens and clone existing components (`BudgetCard`, `SegmentedControl`, `TelemetryUnavailableNotice`) — never hand-roll new styling.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (hand-rolled CSS design tokens via Tailwind v4 `@theme`; no shadcn, no `components.json`, no `tailwind.config.*`) |
| Preset | not applicable |

**shadcn gate:** N/A. The project predates shadcn with a locked, hand-rolled token system (`index.css` header explicitly states "Single source of truth for color, spacing, typography. Locked by UI-SPEC"). Initializing shadcn would fork the design language. Registry safety gate is therefore not applicable.

**Theme:** Dark default (`--color-surface-base: #0b0f14`), light-theme opt-in via `<html class="light-theme">`. The new panel and selector use design-token variables only, so they theme automatically — no per-theme code.

## Spacing Scale

Declared values (INHERITED from `index.css` `@theme` — 8-point base; do not introduce new spacing primitives):

| Token | Value | Usage |
|-------|-------|-------|
| `--spacing-1` | 4px | Sub-label gap under each trio stat; figure-to-caption gap (matches BudgetCard `marginBottom: 4px`) |
| `--spacing-2` | 8px | Segmented-control button padding (`4px 8px`); inter-stat gap inside the trio on compact widths |
| `--spacing-3` | 12px | (available; reserved) |
| `--spacing-4` | 16px | Panel inner padding (`p-4`); grid gap between panels (`gap-4`); toolbar gap |
| `--spacing-6` | 24px | (available; section padding) |
| `--spacing-8` | 32px | (available) |
| `--spacing-12` | 48px | (available) |
| `--spacing-16` | 64px | (available) |

Exceptions:
- **Sparkline height: 48px** (`--spacing-12`). A glanceable hit-rate trend strip, not a full chart. Width fills the panel content box (100%). This is on-scale (48 = 12 × 4px base) and matches the "small sparkline" intent of D-05.
- **Trio stat min-height:** the trio row reuses BudgetCard's `flex flex-col gap-1 rounded border p-4` container; the panel as a whole keeps the `min-h-28` floor that the existing `TelemetryUnavailableNotice` uses, so a degraded panel occupies the same footprint as a populated one (no layout shift).

## Typography

| Stat caption | 12px | 600 | 1.4 (`--text-label`) | The lowercase/uppercase sub-label under each figure ("hit", "creation", "saved"). Mono, muted (`--color-text-muted`), matches BudgetCard's `<h3>` label treatment (uppercase, `letterSpacing: 0.05em`). |

## Color

INHERITED 60/30/10 split from `index.css` `@theme`. The cache-efficiency panel is a neutral surface like every other TelemetryView panel — it does NOT consume the 10% accent.

| Role | Value (dark / light) | Usage in this phase |
|------|----------------------|---------------------|
| Dominant (60%) | `--color-surface-base` `#0b0f14` / `#ffffff` | TelemetryView scroll background behind the panel grid |
| Secondary (30%) | `--color-surface-raised` `#161b22` / `#f6f8fa` | The cache-efficiency panel card background; the trio container (clones BudgetCard) |
| Accent (10%) | `--color-accent` `#06b6d4` / `#0891b2` | **NOT used by this panel's content.** Reserved exclusively for the 6-item list below. The sparkline does NOT use accent. |
| Destructive | `--color-destructive` `#f85149` / `#dc2626` | **NOT used.** No destructive action exists in this phase (read-only observability). |

**Sparkline stroke/fill:** use a neutral chart color, NOT accent. Match the existing single-series chart palette already used by ChartPanel (`palette-1` / the non-failure default series color the existing panels render). The hit-rate trend is informational, not a status signal — do not color it green/red.

**Accent reserved for (6-item explicit list, INHERITED from `index.css` header — unchanged this phase):**
1. Primary CTA button background
2. Selected-node ring
3. Currently-active wave-band edge highlight
4. SSE-connected status dot
5. Focus ring outline on keyboard focus (`:focus-visible` — applies to the new segmented-control buttons automatically)
6. Connection-status pill background (state=connected only)

The cache-efficiency panel, the trio figures, the sparkline, and the per-level selector are NONE of these — they stay on neutral surface/text/border tokens. The only accent that touches the new selector is the inherited `:focus-visible` ring (#5), which is automatic and correct.

**Status-token note (D-04 enforcement):** Per D-04, render hit-ratio, cache-creation, and savings as RAW values. Do NOT apply `--color-status-warning`/`--color-status-error` to a low hit-ratio — that would editorialize "caching broken." Trio figures use `--color-text-primary` regardless of value. No inline caveat, no color-coded judgment.

## Copywriting Contract

This is a read-only observability panel. There is no primary CTA and no destructive action — the contract below covers the panel's labels and its empty / loading / degraded / zero-cache states, which ARE the genuinely-unanswered micro-decisions for this phase.

| Element | Copy |
|---------|------|
| Primary CTA | none (read-only dashboard — DASH-05; no mutation paths) |
| Panel title | `Cache Efficiency` (rendered uppercase via the existing label CSS) |
| Trio stat 1 caption | `hit` (figure: hit ratio %, e.g. `11.9%`) |
| Trio stat 2 caption | `creation` (figure: cache-creation tokens, formatted like Token Breakdown — `12.3k` / `1.2M`) |
| Trio stat 3 caption | `saved` (figure: realized savings $, via `formatCents`, e.g. `$0.04`) |
| Sparkline caption | `hit-rate over time` (12px mono muted, below the strip) |
| Empty state (Prometheus configured, zero data in window) | Figures render `0%` / `0` / `$0.00`; sparkline renders flat-at-zero. Caption beneath: `No cache activity in this window`. Do NOT show a dashed-border notice for zero-data — only for genuine unavailability. |
| Zero-cache / divide-by-zero (hit ratio `0/0` → NaN) | Display `—` for the hit-ratio figure (per Research Pitfall 4). The `creation` and `saved` figures still render their real values. No error styling. |
| Loading state | Reuse the existing panel loading affordance (the trio figures show a muted placeholder until the first range fetch resolves; match how ChartPanel renders pre-fetch). No spinner unless the existing panels use one. |
| Error state (Prometheus unavailable — `{"status":"unavailable"}` HTTP 200, OR HTTP 502 unreachable) | Reuse `<TelemetryUnavailableNotice>` verbatim. Default message `Telemetry unavailable — Prometheus not configured`; for 502 pass a message containing the word `unreachable`. Same degradation contract as the four existing panels. |
| Per-level selector option labels | `Project` (the existing default, = no breakdown), `Phase`, `Plan`, `Wave`. Level options render in `--font-mono` (`mono: true`), matching how the scope toggle monos the project name. |
| Destructive confirmation | none (no destructive actions in this phase) |

**D-04 reminder for copy:** No footnote, asterisk, or tooltip explaining the Phase-20 CLI-scaffold cache reality. The numbers stand raw. The only explanatory copy permitted is the value-neutral `No cache activity in this window` empty-state line.

## Component & Interaction Inventory

New surfaces this phase. Each clones an existing component — no net-new primitives.

| Surface | Clones / Reuses | Placement | Behavior |
|---------|-----------------|-----------|----------|
| **Cache-Efficiency panel** (D-05) | `BudgetCard` container shell (`flex flex-col gap-1 rounded border p-4`, `--color-surface-raised` bg, `--color-border-subtle` border) for the trio; existing ChartPanel/`TimeSeriesChart` (recharts `AreaChart`) for the sparkline | Inside the existing `data-testid="telemetry-panels"` grid (`grid grid-cols-2 gap-4`), appended after the four current panels. Occupies one grid cell. | Trio = three `increase(counter[$w])` range sums (hit ratio derived in PromQL: `cache_read / (cache_read + cache_creation)`); sparkline = hit-ratio range series over the selected window. Polls on the existing 60s cadence via the existing `query_range` proxy. `data-testid="panel-cache-efficiency"`. |
| **Per-level selector** (D-06) | `SegmentedControl` (existing, lines 701–743) | Toolbar, placed BETWEEN the scope toggle (left) and the range selector (right) — i.e. left-aligned next to scope, so the toolbar reads `[scope] [level]  …  [range]`. Keep the existing `flex items-center justify-between` toolbar; group scope+level in a left `flex gap-2` cluster, range stays right. | Sets `breakdownDimension: "none" \| "phase" \| "plan" \| "wave"`. When `!= "none"`, ALL panel queries (existing four + new) switch their aggregation to `sum by(<dim>)(...)`, exactly as the scope toggle switches all panels at once. `data-testid="telemetry-level-selector"`. Default `none` = current behavior unchanged. |

**Interaction notes:**
- The per-level selector is the ONLY new control; it is value-bounded (TypeScript enum literals `phase`/`plan`/`wave`), so no free-form text reaches PromQL (no injection surface — matches Research §Security V5).
- Changing the level re-derives every panel's series the same way changing scope does today; the existing key-derivation in `fetchPanel` (reads `matrix.metric["project"]`) extends to `matrix.metric["phase"|"plan"|"wave"]`.
- The sparkline is non-interactive (no tooltip/crosshair required); if the cloned `TimeSeriesChart` brings a tooltip for free, that is acceptable but not required.
- Keyboard focus on the new segmented buttons shows the inherited 2px accent `:focus-visible` ring (accent-reserved item #5) — no extra work.
