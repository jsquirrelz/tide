## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → BreakdownKind state | User clicks a segmented-control button; the value is a TypeScript enum literal, never a free-form string |
| BreakdownKind → PromQL query string | The breakdown dimension is string-interpolated into `sum by(<dim>)(...)` — injection surface examined below |
| dashboard → /api/v1/query_range proxy | Existing proxy forwards queries verbatim to Prometheus; same surface as before |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-02-01 | Tampering | BreakdownKind → PromQL string interpolation | mitigate | `BreakdownKind` is a TypeScript union type literal `"none" \| "phase" \| "plan" \| "wave"` — not a free-form user input. The `levelOptions` array is a const of that type. TypeScript's exhaustive check prevents any other value from reaching the query string at build time. No user-supplied string is interpolated. |
| T-21-02-02 | Information Disclosure | Cache-efficiency panel exposes token/cost telemetry | accept | Dashboard is already a read-only operator surface (DASH-05). The new panel reads existing counters that are already exposed via the /api/v1/query_range proxy. No new data is surfaced that isn't already queryable by anyone with Prometheus access. |
| T-21-02-03 | Denial of Service | PromQL query complexity (by() aggregation) | accept | The breakdown adds at most one `by(<label>)` clause to existing queries. Label cardinality is bounded by the TIDE project hierarchy (projects/phases/plans/waves); not user-controlled. Same exposure as existing `sum by(project)` queries already issued in scope="all" mode. |
| T-21-02-04 | Denial of Service | Panel count increase from 4 → 5 | accept | One additional query_range call per polling interval (60s cadence). Prometheus is read-only; no write amplification. The cache-efficiency panel uses the same `fetchQueryRange` helper with identical timeout and degradation handling. |
| T-21-02-05 | Tampering | NaN hit-ratio display | accept | PromQL division by zero returns NaN/+Inf; the UI displays "—" (no value). No error path exposes internal state. |
| T-21-02-SC | Tampering | npm/pip/cargo installs | accept | This plan adds zero new npm packages. No package legitimacy gate required. |
</threat_model>
