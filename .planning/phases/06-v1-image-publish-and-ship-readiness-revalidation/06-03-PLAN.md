## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| secrets.GITHUB_TOKEN → ghcr.io push | packages:write scoped to build-images job only; docker/login-action handles token securely (--password-stdin equivalent) |
| v*-rc.* tags → image push | if: !contains(github.ref, '-rc.') guard prevents rc tags from pushing images |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03-01 | Information Disclosure | GITHUB_TOKEN in build-images | mitigate | docker/login-action@v3 handles authentication via stdin, not positional arg; token never appears in run: steps or set -x traces; mirrors --password-stdin pattern from existing chart-publish job |
| T-06-03-02 | Tampering | Partial manifest list (images pushed, chart not yet) | mitigate | chart-publish needs: [build-images, release] — chart is only published after all 6 matrix jobs succeed; a matrix failure leaves chart unpublished (no dangling chart with missing images) |
| T-06-03-03 | Spoofing | packages:write scope on non-image jobs | mitigate | permissions block is job-level on build-images only; helmify-verify, pre-flight keep contents:read; release keeps contents:write (goreleaser needs it for GitHub Release creation) |
| T-06-03-04 | Elevation of Privilege | RC tag triggering image push | mitigate | if: !contains(github.ref, '-rc.') mirrors existing release/chart-publish guard; rc tags only fire helmify-verify and dry-run.yaml |
</threat_model>
