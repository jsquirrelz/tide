    1. **What:** The 6 component images (`tide-controller`, `tide-dashboard`, `tide-stub-subagent`, `tide-credproxy`, `tide-push`, `tide-claude-subagent`) are published to `ghcr.io/jsquirrelz/` as multi-arch (`linux/amd64` + `linux/arm64`) OCI images.

    2. **CI pipeline (IMG-01):** Images publish automatically via the `build-images` matrix job in `.github/workflows/release.yaml` when a `v*` tag is pushed (not for `v*-rc.*` tags). The image tag strips the `v` prefix from the git tag (`v1.0.0` → `1.0.0`) to match the Helm chart `appVersion`. Run order: `helmify-verify` → `build-images` (parallel with `pre-flight`) → `chart-publish` (after both `build-images` and `release`) — images are always pushed before the chart.

    3. **Local pre-publish path (IMG-LOAD-01):** Before cutting a tag, the operator can verify the full BOOT-04 path at $0 cost using locally-built images:
       ```
       make acceptance-v1-smoke
       ```
       This runs `ACCEPTANCE_SAMPLE=small hack/scripts/acceptance-v1.sh`, which calls `hack/scripts/load-images-if-needed.sh` to probe each image via `docker manifest inspect ghcr.io/jsquirrelz/<name>:1.0.0`; if absent, builds locally and `kind load docker-image`s into the cluster. No API key required.

    4. **GHCR visibility (Pitfall 3):** The first push of each new package to `ghcr.io/jsquirrelz/` defaults to **private**. After the first successful `build-images` CI run, the maintainer must navigate to `https://github.com/users/jsquirrelz/packages/container/<name>/settings` for each of the 6 images and set visibility to **Public**. Until this is done, `docker manifest inspect` returns 401 even though the image exists.

    5. **Existence check:** `docker manifest inspect ghcr.io/jsquirrelz/tide-controller:1.0.0` — exits 0 if published and public; non-zero if absent or private.
