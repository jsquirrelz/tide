## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Anthropic API → credproxy | Upstream responses cross into classification logic |
| Subagent pod → reconciler | EnvelopeOut.Reason (termination-message channel) is produced inside the workload pod |
| Operator CLI → Project status | tide resume writes the status subresource |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-04 | Tampering | Forged billing reason in EnvelopeOut (malicious/compromised subagent fakes "credit balance") | accept | Worst case is self-DoS: dispatch halts on the attacker's own project; no privilege escalation, no data exposure; operator recovers with `tide resume`. Envelope already HMAC/controller-spawned-pod gated. |
| T-13-05 | Tampering | credproxy response-classification tampering (subagent forging an upstream 400 to trip the latch) | accept | The latch lives in the proxy process, keyed off genuine upstream responses over verified TLS to api.anthropic.com; the subagent cannot inject upstream responses. A subagent crafting a request that legitimately yields a billing 400 implies the balance IS exhausted — halt is the correct outcome. |
| T-13-06 | Denial of Service | Anthropic rewording the billing message → missed halt | mitigate | Conservative "credit balance" substring (all observed variants); failure mode degrades to pre-Phase-13 behavior (sessions fail individually), never a crash. |
| T-13-07 | Denial of Service | Over-broad classification halting on unrelated 400s | mitigate | Classifier requires status 400 AND the billing-specific substring; unit tests pin false-positive cases (invalid model, cap-hit, forced-failure). |
| T-13-08 | Elevation of Privilege | tide resume clearing conditions it shouldn't | mitigate | resumeRun removes exactly ConditionBillingHalt via meta.RemoveStatusCondition; RBAC for projects/status update is the existing operator grant — no new permission added. |
| T-13-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase (zero new dependencies). |
</threat_model>
