## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Project CR author → controller | spec.subagent.image / levels.<level>.image are untrusted input that becomes a Job container image ref |
| Helm operator → controller | CLAUDE_SUBAGENT_IMAGE env / --subagent-image flag set by cluster operator (trusted) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Elevation of Privilege | resolveImage → BuildJobSpec (arbitrary image dispatch via Project spec) | accept | Project create/update RBAC already implies agent code-execution against the target repo (prompt + repo + creds are in the same spec); the dispatched image runs under the same constrained tide-subagent SA, namespace-scoped, behind credproxy — no privilege gained beyond what Project authorship already grants. Image allowlisting is a future hardening item, out of v1.0.1 scope. |
| T-13-02 | Tampering | Image ref string injection (shell/YAML metachars in image field) | mitigate | Image ref is passed as a typed `Container.Image` field via client-go (no shell interpolation); kubelet rejects malformed refs. Regression tests assert the ref is byte-identical pass-through. |
| T-13-03 | Denial of Service | Empty resolved image (no flag, no env, no CRD field) → Job with empty image | mitigate | resolveImage doc contract: empty return means unconfigured; helm default ships non-empty (`subagent.defaults.image` after 13-03; flag/env before). Existing kubelet behavior fails the pod with ImagePullBackOff/InvalidImageName — observable, not silent. |
| T-13-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase (RESEARCH Package Legitimacy Audit: N/A — zero new dependencies). |
