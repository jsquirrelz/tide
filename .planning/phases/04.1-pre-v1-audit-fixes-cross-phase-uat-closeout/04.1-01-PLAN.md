<objective>
Pre-empt the most common Phase 02.2-style cascade class — "missing prerequisite surfaces 3 plans in" — by running every Wave 5/6/7 UAT environmental check NOW, before any source-fix plan dispatches. Phase 02.2 ran 12 plans to close one timing fix because Docker/kind/helm-replace/image-load gaps surfaced one cascade at a time. This Wave 0 plan finds all of them at once.

The output is a single markdown report (`04.1-01-READINESS.md`) that downstream UAT plans (12/13/14) read at start. Each tool either:
- PASS — version + path documented, UAT plan can use it directly
- FALLBACK — tool absent but a documented manual alternative exists (e.g., Chrome DevTools MCP unavailable → manual screenshots per RESEARCH.md §UAT Closeout)
- BLOCKED — tool absent with no fallback; UAT plan downstream is BLOCKED until operator installs (this is the only outcome that halts the chain)

This is a checkpoint plan — executor runs the checks, writes the report, and surfaces any BLOCKED rows to the human for an install-and-resume decision. Source-fix plans (Wave 1 onward) DO NOT depend on Wave 0 (they need only `go build` / `make test`, which is assumed); only Wave 5/6/7 UAT plans depend on this.

Purpose: prevent cascade-by-cascade discovery of tooling gaps during live UAT runs.
Output: `04.1-01-READINESS.md` committed to the phase directory.
</objective>

<task id="1" wave="0" name="Run tooling readiness probes and record results" depends_on="[]">
<action>
Run each readiness probe in order, capturing exit code + stdout snippet for each. Write the consolidated results to `.planning/phases/04.1-pre-v1-audit-fixes-cross-phase-uat-closeout/04.1-01-READINESS.md` using the report template below.

Probes (in order):

1. **Docker daemon:**
   - `docker info --format '{{.ServerVersion}}'` — non-empty stdout, exit 0
   - On failure: BLOCKED. Operator must start Docker Desktop / `dockerd`.

2. **kind binary:**
   - `kind version` — stdout matches `kind v0.31.` prefix
   - On version mismatch: FALLBACK with note "v0.31.x is the empirically-closed Phase 02.2 baseline; other versions may surface cascades"
   - On missing binary: BLOCKED. `brew install kind` or `go install sigs.k8s.io/kind@v0.31.0`.

3. **kubectl:**
   - `kubectl version --client -o yaml | grep gitVersion` — non-empty
   - On missing: BLOCKED. `brew install kubectl`.

4. **helm v3:**
   - `helm version --short` — stdout matches `^v3.`
   - On missing: BLOCKED. `brew install helm`.

5. **gsd-sdk CLI + audit-uat JSON shape:**
   - `gsd-sdk query audit-uat` returns valid JSON
   - AND the JSON shape matches what Plans 12-15 parse:
     ```bash
     AUDIT_PROBE=$(gsd-sdk query audit-uat 2>&1)
     echo "$AUDIT_PROBE" | python3 -c "
     import json, sys
     d = json.load(sys.stdin)
     assert 'summary' in d, 'audit-uat output missing summary key'
     assert 'results' in d, 'audit-uat output missing results key'
     assert 'total_items' in d['summary'], 'audit-uat summary missing total_items'
     assert 'by_phase' in d['summary'], 'audit-uat summary missing by_phase'
     print('audit-uat schema OK: total_items=' + str(d['summary']['total_items']))
     "
     ```
   - The probe fails fast (non-zero exit) on schema mismatch — this catches the case where the SDK shim changes its output format before any Wave 6-9 UAT plan dispatches and discovers it mid-run.
   - On non-zero exit OR missing keys: BLOCKED. gsd-sdk shim under `~/.local/bin/` per RESEARCH.md sources; if the SDK is present but the schema mismatches, surface the actual JSON shape in the readiness report so Plans 12-15 can be re-plumbed before dispatch.

6. **Chrome DevTools MCP:**
   - Probe MCP availability via the executor's MCP listing. If the executor cannot detect Chrome DevTools MCP, mark as FALLBACK with note "Wave 7 item 3 (dashboard live verification) will downgrade to manual-screenshot UAT per RESEARCH.md §UAT Closeout — Wave 7."
   - Do NOT BLOCK on missing Chrome DevTools MCP — manual screenshots are an acceptable documented fallback.

7. **Image availability (preload check, advisory only):**
   - `docker image ls --format '{{.Repository}}:{{.Tag}}' | grep -E '^(controller|stub-subagent|credproxy|tide-push|claude-subagent):' || echo "NO LOCAL IMAGES — will be built by Wave 5+ via make test-int-kind-prep"`
   - This row is INFO only. `make test-int-kind-prep` is invoked by Wave 5 (Plan 12); do not preload here.

8. **`ANTHROPIC_API_KEY` presence (Wave 6 item 2 prerequisite):**
   - `[ -n "${ANTHROPIC_API_KEY:-}" ] && echo "SET" || echo "UNSET"`
   - On UNSET: FALLBACK with note "Wave 6 item 2 (live Claude E2E) will defer to a documented cost-acknowledgement checkpoint; if operator declines, item flips to 'deferred-to-Phase-5' status."
