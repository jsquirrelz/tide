# Phase 04.1 Plan 08: TaskReconcilerDeps Carrier Struct Summary

Pure mechanical refactor consolidating `TaskReconciler`'s 8 dispatch-tier fields into a `TaskReconcilerDeps` carrier struct, mirroring the `HelmProviderDefaults` precedent already used by Milestone/Phase/Plan reconcilers.

## What Was Done

Declared `type TaskReconcilerDeps struct` in `internal/controller/task_controller.go` carrying 8 dispatch-tier fields:

| Field | Type |
|-------|------|
| Dispatcher | dispatch.Dispatcher |
| Budget | *budget.Store |
| Defaults | budget.Limits |
| SigningKey | []byte |
| SubagentImage | string |
| CredproxyImage | string |
| EnvReader | podjob.EnvelopeReader |
| Recorder | record.EventRecorder |

`TaskReconciler` struct was updated to replace those 8 top-level fields with a single `Deps TaskReconcilerDeps` value field. All 12 method-body references inside `task_controller.go` updated to use `r.Deps.<Field>` form.

**Fields that stayed direct (not moved to Deps):** `PlannerPool`, `ExecutorPool`, `WatchNamespace` — they are concurrency limiters/scope controls, not dispatch-tier deps.
