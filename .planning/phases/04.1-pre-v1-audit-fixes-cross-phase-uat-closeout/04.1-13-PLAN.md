# RWX Storage Driver Matrix

TIDE's `tide-projects` PersistentVolumeClaim is `ReadWriteMany` by default
(see `charts/tide/templates/projects-pvc.yaml`) so multiple Pods can mount
the same workspace volume concurrently — the Phase 2 dispatch contract
depends on this for envelope-writer-init + subagent-main + credproxy-sidecar
to share `/workspace`.

The chart sets `storageClassName: ""` (empty), delegating driver choice to
the cluster's default StorageClass. For production deployments, operators
should explicitly set a driver that supports RWX. The table below covers
the five drivers TIDE has been tested against (or known to work via the
upstream driver's RWX support).

| Driver | Cloud | Access Modes | Provisioning | Bandwidth Class | Notes |
|--------|-------|--------------|--------------|-----------------|-------|
| AWS EFS (`efs.csi.aws.com`) | AWS | RWO + RWX | Dynamic + Static | Bursting/Provisioned | Regional; cross-AZ supported. Slower than EBS but the only RWX option in AWS for general workloads. |
| GCP Filestore (`filestore.csi.storage.gke.io`) | GCP | RWO + RWX | Dynamic | Basic/Premium/Enterprise | Regional or zonal. Premium ≥ 2.5TB. Native GKE integration. |
| Azure Files (`file.csi.azure.com`) | Azure | RWO + RWX | Dynamic + Static | Standard/Premium | Cross-AZ supported. SMB or NFS protocol; NFS recommended for POSIX semantics. |
| csi-driver-nfs (`nfs.csi.k8s.io`) | Any | RWO + RWX | Dynamic via NFS server | Depends on backing NFS server | Bring-your-own NFS. Good for on-prem; requires external NFS server. |
| Longhorn (`driver.longhorn.io`) | Any | RWO + RWX (with NFS) | Dynamic | Depends on backing disks | Cloud-native distributed storage. RWX requires the Longhorn NFS feature; otherwise RWO only. |

## Setting the StorageClass

In Helm values:
```yaml
storageClassName: "efs-sc"  # or filestore-csi-rwx, azurefile-csi, etc.
```

Or via direct PVC patch (if installed without re-templating):
```bash
kubectl patch pvc tide-projects -n <namespace> -p '{"spec":{"storageClassName":"efs-sc"}}'
```

# Expected: STATUS = Bound
```

If the PVC stays `Pending`, check the StorageClass exists in-cluster:

```

And inspect the driver's CSI controller pod logs for provisioning errors.
