# Phase 04.1 Plan 10: P4.2 Project-Driven Credproxy Allowlist Summary

JWT-independent, env-var-driven credproxy upstream allowlist made configurable via Project-level `Spec.Providers[*].AllowedRoutes` with admission-time denylist enforcement for admin/billing paths.

## What Was Built

**6-tier implementation of P4.2 (ROADMAP SC#9):**

1. **Schema** (`api/v1alpha1/project_types.go`): New `RouteSpec{Method, PathPrefix}` struct with CEL validation (`Method Enum=GET;POST`, `PathPrefix Pattern=^/v1/...`). `ProviderConfig.AllowedRoutes []RouteSpec` field added (optional, `omitempty`). Codegen regenerated via `make manifests` + `make generate`.

2. **Webhook validation** (`internal/webhook/v1alpha1/project_webhook.go` — NEW): `ProjectCustomValidator` rejects `PathPrefix` matching `/v1/admin` and `/v1/billing` (including subpaths) at admission. Hardcoded denylist; cannot be overridden by operator config. Defense-in-depth layer against the runtime credproxy.

3. **Sidecar code** (`internal/credproxy/server.go`): Added `RouteSpec` struct (package-private, intentionally duplicated from api/v1alpha1 per import-firewall rule). Added `Proxy.ExtraAllowedRoutes []RouteSpec` field. Promoted `isAllowedRoute` from package-level function to `*Proxy` method — checks hardcoded baseline first, then ExtraAllowedRoutes. Hardcoded baseline (`POST /v1/messages`, `POST /v1/messages/count_tokens`) is preserved verbatim.

4. **Sidecar wiring** (`cmd/credproxy/main.go`): Parses `TIDE_ALLOWED_ROUTES` env var (JSON-encoded `[]RouteSpec`). Empty/unset → empty slice → baseline-only. Populates `Proxy.ExtraAllowedRoutes`. Logs count+routes at INFO on startup.

5. **Dispatch wiring** (`internal/dispatch/podjob/jobspec.go`): Added `marshalAllowedRoutes` helper that flattens `Project.Spec.Providers[*].AllowedRoutes` into a JSON array (returns `"[]"` on empty). `TIDE_ALLOWED_ROUTES` env var injected on the credproxy container alongside `TIDE_TASK_UID` and `TIDE_PROXY_LISTEN`.

6. **Manager wiring** (`cmd/manager/main.go`): `SetupProjectWebhookWithManager(mgr)` wired alongside existing Plan and Wave webhook setups at `:449`.

## Key Design Decisions

- **Schema additive only**: `AllowedRoutes` is a new optional field — no renames, no breaking changes to existing Projects.
- **Import firewall preserved**: `RouteSpec` lives in BOTH `api/v1alpha1` (CRD schema) AND `internal/credproxy` (runtime use). JSON serialization at the env-var boundary bridges them. `internal/credproxy` does NOT import `api/v1alpha1` — providerfirewall analyzer confirmed passing.
- **charts/tide/values.yaml UNTOUCHED**: `AllowedRoutes` is a per-Project Spec field, not a chart key. PIT-1 (chart is FIXED contract) honored.
- **config/webhook/manifests.yaml updated**: `make manifests` regenerated the project webhook registration entry — expected and intentional.
- **Denylist is primary gate**: webhook rejects admin/billing at admission; credproxy enforces at runtime. Both must agree — webhook is primary; credproxy is fail-safe.
