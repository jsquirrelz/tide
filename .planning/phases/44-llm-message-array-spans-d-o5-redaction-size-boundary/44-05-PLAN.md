<objective>
Close MSG-01's manager side: Tasks — which have never had a reporter Job — gain a D-06-gated trace-only reporter spawn at both terminal call sites in `handleJobCompletion`, keyed per-attempt on the completed Job's UID, carrying the Task's own span ID as `--traceparent` so its LLM message-array spans nest under the Task AGENT span Phase 43 shipped. Failed Tasks are covered identically (D-05 — a failed Task's conversation is the highest-value debugging trace). Failed planner Jobs need no new spawns: the existing four planner sites already spawn reporters on every terminal Job, and 44-04's synth-before-materialization ordering makes those runs emit conversation spans regardless of envelope state.

**EXECUTION GATE (phase-wide):** requires `worktree-phase-43-discuss` merged into `main` first. This plan compiles against 43-05's `emitTaskSpanOnce` (which mirrors `task.Status.TaskTraceSpanID` in-memory unconditionally after emission — the value this plan's spawn reads in the same reconcile) and 43-03's `traceparentForLevel`. It also requires 44-02 (ReporterOptions.TraceOnly/OTLPEndpoint) and 44-04 (the --trace-only flag registered — spawning an Arg the binary doesn't register would crash-loop every reporter Job; Phase 43 Pitfall 4).

Purpose: this is the plan that makes the milestone's headline capability actually fire in a cluster.
Output: Task trace-only spawns wired + manager deps + envtest proof.
</objective>
