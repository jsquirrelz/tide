<objective>
Ship the OpenTelemetry foundation — a thin `pkg/otelai` attribute-helper package (D-O4) and `internal/otelinit` TracerProvider lifecycle wiring (D-O3) — and connect both to the orchestrator's `cmd/manager` boot path. This plan creates the seam that reconciler edits in plans 04-05/04-06 will call to emit OpenInference-attributed spans across the dispatch chain (Project → Milestone → Phase → Plan → Task).

Purpose: TIDE has no Go OpenInference SDK to depend on (none exists in 2026 — D-O4), so we hand-roll exactly five attribute helpers matching the spec's flat-key encoding. TracerProvider construction MUST omit `WithSampler(...)` so that `OTEL_TRACES_SAMPLER` env var (default `parentbased_traceidratio` arg `0.1` from chart values) is honored (Pitfall 24). When `OTEL_EXPORTER_OTLP_ENDPOINT` is empty the system degrades to a no-op so the orchestrator works in `kind` clusters without an OTLP collector.

Output: `pkg/otelai/{attrs.go,attrs_test.go,doc.go}` + `internal/otelinit/{provider.go,provider_test.go,doc.go}` + cmd/manager wiring; new go.mod entries for otel/sdk/trace, otel/exporters/otlp/otlptrace/otlptracegrpc, otel/trace/noop pinned to v1.43.0.
