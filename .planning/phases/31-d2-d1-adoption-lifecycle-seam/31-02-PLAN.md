<objective>
Implement D2: an adopted Project advances `Initialized → Running` once
`ImportComplete=True` and the import tree is confirmed present, WITHOUT
dispatching a project-planner Job — and durably suppresses any future
re-dispatch via a `.status` condition that survives a cold-cache restart. The
lifecycle advance is what unblocks D1 (the budget gate only evaluates once
`Phase==Running`); D1's enforcement on the adoption path is verified here via
the ACTUAL halt mechanism (ConditionBudgetBlocked → dispatch gate refuses),
while D1's child-level idempotency lands in plan 03.

Purpose: Closes the "spent blind" headline failure at its shared call site and
removes the cache-as-truth anti-pattern (live List → durable condition).
Output: Modified `reconcileProjectPlannerDispatch`; new envtest file.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| reconcile goroutine → CRD `.status` | The adoption arm writes Phase + condition; concurrent reconciles of the same Project can race on resourceVersion. |
| controller → PlannerPool | A mis-ordered return could acquire a pool slot it never releases. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-03 | Tampering / race | Phase-advance + suppression stamp | mitigate | D-07: batch both mutations into ONE Status().Patch(MergeFrom(base)); MergeFrom optimistic concurrency surfaces conflicts as a retryable err rather than a lost update. |
| T-31-04 | Denial of Service (slot leak) | PlannerPool.Acquire at L1137 | mitigate | D-06: both new returns occur before Acquire (verified by acceptance criterion line-ordering check); preserves the L1079 documented invariant. |
| T-31-05 | Denial of Service (retry storm) | suppressed/halt return paths | mitigate | D-08: expected/permanent states return ctrl.Result{}, nil (or RequeueAfter), never a Go err — mirrors billing_halt.go / failure_halt.go. |
| T-31-06 | Spoofing (stale-cache re-dispatch) | informer-cache miss after restart | mitigate | D-01/D-02: authoritative short-circuit reads the durable `.status` condition before the live List, so a cold cache cannot resurrect a paid project-planner dispatch (ADOPT-05 envtest). |

No new external input boundary is introduced. All threats are
controller-internal correctness/availability and are mitigated by the locked
decisions D-06/D-07/D-08; the ADOPT-05 envtest is the regression guard for T-31-06.
