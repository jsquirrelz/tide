(default .tgz, `--dir` for unpacked) using pkg/bundle from 29-01.

round-trips. Satisfies ROADMAP criterion #1.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator kubeconfig → cluster | export reads CRs + creates an inspector pod under the operator's RBAC |
| PVC bytes → CLI | envelope bytes tarred out of the per-namespace PVC are read into the local bundle |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-02-01 | Elevation | inspector pod PVC access | mitigate | PVC mounted ReadOnly with SubPath=projectUID — pod cannot reach other projects' subtrees or write (severity: medium) |
| T-29-02-02 | Information disclosure | RBAC scope | mitigate | Verbs limited to pods create/get/delete + pods/log get in the target namespace (document in the RBAC comment block); no cluster-wide read (severity: medium) |
| T-29-02-03 | Tampering | shell injection via tar args | mitigate | tar command is a fixed string with no user-controlled path interpolated (unlike artifact-get's ARTIFACT_PATH); no `sh -c` with operator input (severity: low) |
| T-29-02-04 | Spoofing | wrong oldUID in seed | accept | UIDs are read from live CRs at export time, not external input (RESEARCH security table) (severity: low) |
</threat_model>
