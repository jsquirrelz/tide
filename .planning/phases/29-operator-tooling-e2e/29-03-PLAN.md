<objective>
Implement `tide import-envelopes <bundle>` with both modes: `--dry-run` (offline preview — no
cluster, no pods, reuses the 29-01 validator) and live stage-only mode (loader pod streams the
tgz onto the PVC at old-UID paths, creates the seed ConfigMap, surfaces project.yaml). It does
NOT apply the Project — the operator runs `tide apply project.yaml` separately (D-05).

Purpose: The consumption half of the round-trip. Dry-run gives the operator a preview before
committing (criterion #2); live mode stages so a subsequent Project apply triggers Phase 28's
adoption (criterion #3). Satisfies ROADMAP criteria #2 and #3.

test/integration/kind/loader_exec_smoke_test.go (the live-cluster SPDY-exec gate for A1/A2, so the

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| bundle .tgz → loader pod PVC | attacker-influenceable tgz is unpacked into the per-namespace PVC; must not escape pvcSubPath |
| bundle → seed ConfigMap | a crafted bundle could declare CRs/children the operator did not intend |
| operator kubeconfig → cluster | loader pod + ConfigMap created under operator RBAC (now needs pods/exec) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-03-01 | Tampering | loader pod tgz unpack | mitigate | Loader unpacks to a fixed `-C /workspace`; the CLI bundle reader (pkg/bundle, 29-01) already rejects `../`/absolute entries before they reach the pod; pod has no shell-eval of entry names (severity: high) |
| T-29-03-02 | Elevation | loader pod PVC write scope | mitigate | RW mount confined by SubPath=<oldProjectUID>/workspace in the target namespace only; no cross-namespace mount, no absolute-path write (Phase 28 D-08 confinement preserved) (severity: high) |
| T-29-03-03 | Spoofing/Tampering | crafted seed/ConfigMap injection | mitigate | Dry-run + Phase 28 Kind-allowlist + name-match are the adoption gates; operator-gated apply (D-05) means no unverified envelope auto-reaches client.Create; import is stage-only (severity: high) |
| T-29-03-04 | Tampering | out.json integrity | mitigate | sha256 verified in dry-run (29-01) before the operator commits; mismatch → re-plan, never adopt (D-04) (severity: medium) |
| T-29-03-05 | Elevation | RBAC for pods/exec | mitigate | Loader pod needs pods create/get/delete + pods/exec create (new verb beyond inspector); document minimal verbs in the RBAC comment block; scoped to target namespace (severity: medium) |
| T-29-03-06 | Repudiation | import without preview | accept | Dry-run is offered but not forced; operator-gated flow (dry-run→review→apply) is the documented path (severity: low) |
| T-29-03-07 | Tampering | untested SPDY exec path | mitigate | Task 3's live kind smoke exercises the real pods/exec URL+verb against a real apiserver, so a wrong construction cannot reach 29-05/production silently — the A1/A2 assumption is gated in-plan (severity: medium) |
</threat_model>
