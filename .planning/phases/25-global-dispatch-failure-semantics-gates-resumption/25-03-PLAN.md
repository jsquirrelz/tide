<objective>
Deliver the conservative failure profile (DISP-02). Strict semantics already fall out of the global indegree model from 25-02 (a failed task is never Succeeded, so its dependents never reach indegree 0 while its independent non-dependents keep dispatching). This plan adds the one net-new mechanism: a project-wide `ConditionFailureHalt` that mirrors `BillingHalt`, stamped on the first task execution failure under conservative profile, gating the four execution dispatch sites, and cleared by `tide resume --retry-failed`. It also closes two inherited debts: the missing wave-prune in-flight guard and confirming the strict-profile contract.

Purpose: The spec's wave-boundary contract requires "non-dependents dispatch in strict / halt in conservative". Strict is free; conservative needs the halt rail. Reusing the proven BillingHalt pattern (five-site gate, project condition, cleared by resume) keeps operator muscle memory identical and avoids a bespoke stateful failure tracker.
Output: `failure_halt.go`; `checkFailureHalt` at four execution dispatch sites; `setFailureHaltIfNeeded` in `handleJobCompletion`; `tide resume --retry-failed` FailureHalt clear; wave-prune `Status.Phase != "Succeeded"` skip. Turns the DISP-02 strict + conservative RED specs and the resume-clear unit tests from 25-01 GREEN.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| task execution failure → project halt | A failed Job stamps a project-wide condition; a missed stamp (fail-open) means conservative profile keeps dispatching after a failure — the exact spec violation conservative exists to prevent |
| project condition → four dispatch gates | If any one of the four execution sites omits checkFailureHalt, that level dispatches through the halt (fail-open) |
| operator → tide resume --retry-failed | The sole clearer of FailureHalt; clearing on bare resume would silently un-freeze a halted project |
| wave prune → in-flight Wave deletion | Deleting a Running Wave drops observability/coordination for live work |
