## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Task .status (etcd) → dispatch decision | The completed-task set is read from Task CRD status; a stale read or wrong scope could authorize a dispatch whose deps are not actually done |
| changed-Task event → dependent re-enqueue | If the mapper misses a global dependent, that dependent stalls (availability), not a privilege escalation; if it under-resolves coarse refs, a dependent could be authorized early (integrity) |
| ProjectReconciler wave map ↔ TaskReconciler indegree | Two independent computations of the same DAG; disagreement = a task dispatched out of order |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-25-02-01 | Elevation of Privilege (dispatch-authorization integrity) | computeGlobalIndegree coarse-ref resolution | mitigate | Route indegree through the SAME shared resolver as the wave map (Task 1); a coarse ref counts satisfied only when ALL resolved member tasks are Succeeded. Verified by the DISP-01 coarse-ref envtest spec and the depgraph de-dup unit test. Prevents a held/failed member from releasing dependents early. |
| T-25-02-02 | Tampering / scope-confusion | listProjectTasks label selector with empty projectName | mitigate | Assert projectName != "" before the List (Pitfall 2); resolveProject runs earlier in gateChecks and guarantees it.
