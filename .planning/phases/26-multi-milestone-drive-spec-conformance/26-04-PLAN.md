## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dashboard HTTP client → GET /api/v1/projects/{name}/execution-dag | A read-only endpoint serving project-scoped Task/Wave data to the dashboard; {name} is path input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-06 | Information Disclosure | /execution-dag scoping across projects | mitigate | The handler lists Tasks/Waves by client.MatchingLabels{owner.LabelProject: name} and InNamespace(namespace) — results are scoped to the named project; no cross-project leakage. Matches the existing plans.go/projects.go scoping model (no new authz model introduced; dashboard inherits the cluster's existing access boundary). |
| T-26-07 | Tampering / Injection | {name} path param + namespace query param | mitigate | {name} and namespace are used only as label-selector values and an InNamespace scope — passed to the typed K8s client (not string-concatenated into any query/command), so injection is structurally precluded. K8s validates label/namespace values. Default namespace "default" when unset matches existing handlers. |
| T-26-08 | Elevation / Tampering | new route mutating cluster state | mitigate | DASH-05 zero-mutation contract: the route is GET-only and TestZeroMutationRoutes walks the route tree to enforce it. No write path added. |
| T-26-SC | Tampering | npm/pip/cargo installs | accept | No new npm packages (UI-SPEC §Registry Safety: none). `make dashboard-frontend` builds with the existing pinned dependency tree. No `## Package Legitimacy Audit` needed. |

Threat surface is a single read-only, project-scoped GET endpoint plus a read-only React view — proportionate and minimal. No `high`-severity threats.
</threat_model>
