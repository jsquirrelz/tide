<objective>
Fix clone idempotency (SC-1): `pkg/git/Clone` must handle `gogit.ErrRepositoryAlreadyExists` by opening the existing bare repo and fetching from the remote, instead of returning an error. This prevents clone Job retries (backoffLimit=2) from permanently failing on a warm PVC or on a second reconcile after the first clone attempt wrote the repo.

Purpose: The first legitimate task-execution run surfaced `clone failed: repository already exists` even on a freshly-wiped PVC because clone Job retries compound: attempt-1 writes `repo.git`, attempt-2 hits `ErrRepositoryAlreadyExists` and exits non-zero. This blocks the entire task-execution path.

Output: Modified `pkg/git/clone.go` with idempotent clone logic; new tests `TestCloneIdempotent` and `TestFetchAnonymous` in `pkg/git/clone_test.go`.
</objective>
