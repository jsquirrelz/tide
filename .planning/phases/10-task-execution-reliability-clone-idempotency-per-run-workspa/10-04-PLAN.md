<objective>
Fix dashboard project-detail 404 (SC-7): `ProjectsHandler.Get` in `cmd/dashboard/api/projects.go` defaults `namespace = "default"` when `?namespace=` is absent. The frontend calls `/api/v1/projects/medium-project` (no namespace param) after seeing it in the all-namespace List response — this 404s because `medium-project` is in `tide-sample-medium` not `default`. Fix: when `namespace == ""`, do a `client.List` across all namespaces and return the first project whose `Name == name`.

Purpose: The medium sample runs in `tide-sample-medium`; the dashboard list endpoint (which uses all-namespace search by default) shows it, but clicking through to detail 404s. This makes the dashboard useless for observing the medium DoD run. The root cause is a one-line default in Get that conflicts with List's all-namespace behavior.

Output: Modified `Get` in `cmd/dashboard/api/projects.go` with cross-namespace fallback and a shared `buildDetail` helper; new test `TestGetProjectWithoutNamespaceParamFindsAcrossNamespaces` in `cmd/dashboard/api/projects_test.go`.
</objective>
