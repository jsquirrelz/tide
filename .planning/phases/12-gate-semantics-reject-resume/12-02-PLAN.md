## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator CLI → K8s API | resume/approve mutate CR metadata + status with the operator's kubeconfig RBAC |
| --retry-failed → CR status | a status-reset verb that re-triggers spend (reconcilers re-dispatch planner/executor Jobs) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-04 | Elevation of privilege | resume --retry-failed status reset | mitigate | reset gated on explicit flag (deliberate friction, D-06) AND `Status.Phase == "Failed"` only — Running/Succeeded levels are never reset, so the verb cannot duplicate in-flight Jobs or resurrect completed work (Pitfall 3) |
| T-12-05 | Denial of service (spend) | approve as spend-retry | mitigate | D-07 guard: approve refuses Failed levels before writing any annotation, so approval cannot re-fire planners into an empty credit balance |
| T-12-06 | Repudiation | recovery actions | mitigate | every reset stamps a ResumedByUser condition (permanent record on the CR) and prints per-level feedback to the operator |
| T-12-07 | Spoofing | label-based level discovery | accept | findFailedLevel/list walks trust the `tideproject.k8s/project` label; anyone with label-edit RBAC can already redirect every existing CLI verb — existing posture, no new surface |
| T-12-SC | Tampering | package installs | accept | no packages installed by this plan |
