## Public API Locked in This Plan

```go
package dag

// NodeID is the unique identifier of a node in the DAG.
type NodeID = string

// Edge expresses "From must complete before To."
type Edge struct {
    From NodeID
    To   NodeID
}

// CycleError is returned by ComputeWaves when the input graph contains a cycle.
// InvolvedNodes lists every node whose indegree never reached zero, sorted
// lexicographically.
type CycleError struct {
    InvolvedNodes []NodeID
}

func (e *CycleError) Error() string

// ComputeWaves returns the layered topological sort of (nodes, edges).
// Each returned wave is sorted lexicographically for determinism.
// Returns *CycleError if the graph contains a cycle.
// Returns a plain error (NOT a CycleError) if an edge references an
// undeclared node.
// Complexity: O(V + E).
func ComputeWaves(nodes []NodeID, edges []Edge) ([][]NodeID, error)
```

## Decisions Made

- **Test naming dual surface** — both `TestComputeWaves/<Name>` subtests AND `TestComputeWaves_<Name>` functions, delegating to a shared assertion helper. Required because the plan body's acceptance regex `-run TestComputeWaves_AlphaThroughTheta` will not match the subtest path. No duplication of test logic.
- **`DependsOnNonexistent` returns plain error, not `CycleError`** — Phase 2's admission webhook will `errors.As(err, &dag.CycleError{})` to render a cycle diagnostic; an unknown-node error is structurally distinct (it's a schema-validation failure) and should not get the same treatment.
- **`DuplicateEdges` documented as idempotent** — current implementation `indegree[e.To]++` and `succ[e.From] = append(succ[e.From], e.To)` both run per edge, so duplicate edges cancel: indegree increments twice, the source's succ list gets the target twice, so when the source is processed its single processing pass decrements the target twice. Pinned as a test case so future refactors don't break it.
- **Makefile target placement at file bottom** — kept the kubebuilder-default Makefile structure intact and appended the DAG firewall section as its own `##@` group, so `make help` shows it as a distinct category.

The analyzer only inspects the import-string AST (never the imported package's contents) so the stub being functionally empty is correct. Added an inline comment explaining the reason so a future reader doesn't mistake it for an actual K8s dependency.
- **Files modified:** Added `tools/analyzers/dagimports/testdata/src/k8s.io/apimachinery/pkg/runtime/runtime.go`
- **Verification:** `go test ./tools/analyzers/dagimports/...` exits 0; the analyzer fires on the violation fixture and stays silent on the valid fixture.

**2. [Rule 1 - Bug] analysistest scanned the violation fixture's package-doc comment for `// want` directives**
