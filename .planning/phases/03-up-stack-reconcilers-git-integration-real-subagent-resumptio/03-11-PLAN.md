Phase 2 budget infrastructure to consume:
- `Project.Spec.budget.{usdCap, hourlyLimit, ...}` — Phase 2 D-D1..D2 fields. This spec sets usdCap=1.00 in the fixture YAML.
- `Project.Status.budget.{usdSpent, lastUpdated}` — rolled up at Task completion from EnvelopeOut.Usage. This spec asserts post-run.

Build-tag isolation pattern (Go convention):
- First line of `test/e2e/live_claude_test.go`: `//go:build live-e2e` (NO `+build` directive — Go 1.17+).
- Without `-tags=live-e2e`, the file is excluded from compilation; `make test` / `make test-int` never trigger it.
- The Makefile target `test-e2e-live` is the ONLY entry point: `go test -tags=live-e2e ./test/e2e/... -timeout=15m`.

Skip-on-missing-creds (additional defense):
- Inside the spec: `if os.Getenv("ANTHROPIC_API_KEY") == "" { t.Skip("ANTHROPIC_API_KEY not set — skipping live E2E") }`.
- Double-gate: even if a developer accidentally builds with `-tags=live-e2e`, the test self-skips without the env var. No accidental API calls.

Fixture repo choice (Claude's Discretion per checker B4):
- A small Go-template repo snapshot pinned by SHA. Candidate: a clone of `github.com/golang-standards/project-layout` at a fixed commit SHA, OR the simpler approach — host a minimal fixture repo in this repo's `test/e2e/testdata/fixture-repo/` as a tarball that gets extracted + initialized as a real git repo at test setup time.
- The simpler approach is preferred (no external dependency, no host availability concerns, deterministic across CI runs). Document the choice in docs/live-e2e.md.
- The fixture must be small enough that a Claude milestone-planner run consumes <$1.00 worth of tokens — likely <500 LOC across all files.

CI cadence:
- Nightly only via GitHub Actions cron (or whatever CI the user runs).
- Recipe documented in docs/live-e2e.md, NOT wired into the repo's CI directly in this plan (CI wiring is a v1.x infrastructure task, not a Phase 3 code deliverable).
