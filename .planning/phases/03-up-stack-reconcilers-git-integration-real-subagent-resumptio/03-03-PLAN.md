<objective>
Ship the provider-agnostic `pkg/git` package — a thin Go API around `go-git/v5` v5.19.0 covering clone, worktree-add, commit, push (with optional `--force-with-lease`), and fetch. HTTPS+PAT is the default and only auth path (SSH deferred per ART-05 → docs-only mention). Per-host details (GitHub vs GitLab vs Gitea) are absorbed by the generic HTTPS+PAT pattern.

Purpose: All four downstream Phase 3 streams need git operations — the push Job (Stream B: plan 03-06), the orchestrator clone Job (Stream A: plan 03-08), and the Task executor harness worktree-add (out of Phase 3 scope but plumbed). Centralizing in `pkg/git` keeps `pkg/dispatch` and `cmd/tide-push` import-firewall-clean and lets future per-host adapters (`internal/git/gitlab`, etc.) reuse the API.

Output: Six `pkg/git/*.go` files + per-function tests using `httptest` for an in-process bare-repo HTTP server (no network). Go module additions for `go-git/v5@v5.19.0`.
</objective>

Function signatures (D-derived; planner sets exact form):
- `func Clone(ctx context.Context, repoURL, destDir, pat string) (*git.Repository, error)` — bare clone (PlainCloneContext with bare=true).
- `func AddWorktree(repoPath, taskUID, branch string) (string, error)` — returns worktree dir path (research note: go-git/v5 may lack first-class Worktree.Add; if so, use PlainClone-from-bare workaround per RESEARCH.md).
- `func Commit(wt *git.Worktree, msg string, author object.Signature) (plumbing.Hash, error)` — author is per-call (TIDE-bot identity is the caller's concern).
- `func AddPath(wt *git.Worktree, relpath string) error` — stages a single file at `relpath` (relative to the worktree root) for the next commit. Wraps `wt.Add(relpath)`. Consumed by `cmd/tide-push/main.go` (plan 03-06) before Commit per D-B2 / W11 commit-message protocol.
- `func Push(ctx context.Context, repo *git.Repository, branch, lastPushedSHA, pat string) error` — when lastPushedSHA == "" → first push, omit lease; else → ForceWithLease against the SHA.
- `func Fetch(ctx context.Context, repo *git.Repository, pat string) error` — for lease-refresh / clone-update paths.

Auth pattern (every function that touches the remote):
- `Auth: &gitclient.BasicAuth{Username: "x-access-token", Password: pat}`.
- Convention: "x-access-token" is the GitHub idiom; works against GitLab/Gitea generic HTTPS+PAT (verified RESEARCH.md).

Test infrastructure (httptest):
- Spin up an httptest.Server hosting a bare repo via go-git's `transport/server` or by serving the bare repo dir as static HTTP (go-git http-backend pattern).
- All tests bounded by `t.TempDir()` for repo dirs; no network.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller (push Job process) → remote git host | HTTPS+PAT via go-git/v5 transport. PAT is a string parameter (caller's responsibility to source it from envFrom Secret). pkg/git treats it as opaque; never logs it. |
| Local bare repo on PVC ↔ per-Task worktrees | D-B4 worktrees isolation — go-git AddWorktree (or PlainClone-from-file://-bare workaround) provides per-Task index. No cross-Task `.git/index` race. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-301 (Information Disclosure / PAT leak) | pkg/git function parameters | mitigate | PAT is passed as a function arg; pkg/git MUST NOT log it (no fmt.Printf, no slog.With). Verify with grep: `grep -rcE 'pat' pkg/git/*.go` shows only function-arg use, never log calls. Tests confirm function shape; production push Job (plan 03-06) injects PAT from envFrom Secret. |
| T-303 (Stale ForceWithLease overwrites human commits — Pitfall 13) | pkg/git/push.go | mitigate | First-push branch (lastPushedSHA == "") OMITS the lease entirely — go-git refuses to push if remote has a different ref, surfacing the conflict explicitly. Subsequent pushes carry lease against the recorded SHA; if remote diverged, push rejects and the caller (plan 03-06 push Job) emits a structured error that flips Project.Status.phase to PushLeaseFailed. Test 4 of Task 2 exercises this path. |
| T-302 (Push race) | pkg/git/push.go | accept (deferred to caller) | pkg/git is a library; serialization is the caller's contract. Plan 03-06 uses deterministic Job name `tide-push-{project-uid}` (D-B5) to serialize via K8s API AlreadyExists. |
</threat_model>
