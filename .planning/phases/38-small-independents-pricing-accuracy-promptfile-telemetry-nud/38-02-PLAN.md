<objective>
Make the unknown-model pricing fallback observable (COST-02, D-02): a provider-neutral `Usage.PricingFallbackModel` field rides the envelope, and every controller budget-rollup site rolls it up into (a) a sticky, informational `PricingFallbackActive` Project condition naming the unmatched model and (b) a `tide_pricing_fallback_total` Prometheus counter. The stderr warning in the subagent pod stays but is no longer the only signal.

Purpose: today the fallback is one stderr line in a GC'd pod log — the 2026-07-03 run's 2.8× overcount went unnoticed. This plan builds the transport + surface; plan 38-06 (provider side) sets the flag when the post-normalizer lookup misses.
Output: envelope field, condition constants, `pricing_fallback.go` helper, metric, hooks at all seven rollup call sites, envtest coverage.
</objective>
