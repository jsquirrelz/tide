<objective>
Add `--prompt-file <path>` to `tide apply` (PROMPT-01): the CLI reads the file, validates it (size cap, non-empty, single trailing newline trim — D-11), requires the manifest to contain exactly one Project document (D-10), refuses to override an existing `spec.outcomePrompt` (D-09), injects the content, and server-side-applies. CLI-side only — no CRD change; the ConfigMap-ref union (`outcomePromptFrom`) stays a compatible later addition (explicitly out of scope).

Purpose: multi-paragraph outcome prompts are painful to inline in YAML; the first external-repo run hand-edited the manifest.
Output: flag + validation helpers in apply.go, table-driven CLI tests.
</objective>

       - Inject: `unstructured.SetNestedField(prj.Object, content, "spec", "outcomePrompt")`; return the Project object. The kind/apiVersion missing check stays for whichever object is returned.
