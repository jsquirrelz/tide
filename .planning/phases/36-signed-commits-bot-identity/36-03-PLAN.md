<objective>
Inject the resolved agent identity into every pod that commits: the subagent Job (harness task commits) and the push Job (integrate merges + boundary commit, both plan-boundary and wave-integration variants). This is the genuinely net-new mechanism of SIGN-01 — nothing sets the identity env today; the env-read machinery in the binaries (Plan 36-01) has been dead config since v1.0.0. Injection is UNCONDITIONAL (D-03 resolved values are never empty — the compiled default backstops), and planner Jobs receive the env too (harmless; uniform injection is simpler than kind-discrimination).

Purpose: SIGN-01 — the controller-resolved identity actually reaches the three commit sites at runtime.
Output: BuildOptions/PushOptions extensions, both builders injecting env, all six BuildOptions construction sites and both PushOptions sites populated, fixture-backend mirror, builder env-presence tests on BOTH job types.
</objective>

| Boundary | Description |
|----------|-------------|
| controller → Job pod env | Resolved identity crosses from control plane into workload pods (trusted channel; values already admission-validated) |
| subagent pod → git history | LLM-executed code runs in the pod that carries the identity env — env values are public metadata, nothing secret is added |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-36-01 | Tampering | resolved identity → Job env → commit headers | medium | mitigate | Values entering env are admission-validated (Plan 36-02 Pattern markers); builders stamp them as literal EnvVar values (no shell interpolation) |
| T-36-02 | Spoofing | identity env readable/settable inside subagent pod by LLM-executed code | low | accept | A prompt-injected process could re-export the env before committing — but it could equally pass its own -c flags to git; identity is not a security boundary (precedent T-11-01-04). No secret material is added to any pod by this phase. |
| T-36-SC | Tampering | package installs | low | accept | No package-manager installs in this phase (Package Legitimacy Audit: none) |
