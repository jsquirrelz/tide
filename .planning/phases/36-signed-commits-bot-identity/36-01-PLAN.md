<objective>
Create the single shared agent-identity source (`pkg/git/identity.go`) and convert all three commit sites to it, removing tide-push's hardcoded identity. This is the D-04 bot→agent rename (env vars become `TIDE_AGENT_NAME`/`TIDE_AGENT_EMAIL`; compiled-in default becomes `TIDE Agent <tide-agent@tideproject.k8s>`) and the D-05 tide-push fix (env-sourced identity, W11 stability contract preserved). Phase scope is identity-only per D-01/D-02 — no GPG signing code, no signing-key Secret refs, nothing key-related lands here. Per D-07 this plan has no Phase 34 dependency.

Purpose: SIGN-01 — one uniform, configurable identity across harness task commits, integrate merges, and boundary pushes. Today the three sites disagree (tide-push uses a lowercase hardcoded pair; harness/integrate share a different capitalized default) and nothing sets the env they read.
Output: new `pkg/git/identity.go` + tests; all three sites renamed; all pinned-identity tests updated; repo-wide grep gate proving the old names are gone from Go code.
</objective>
