<objective>
Close DEBT-04 (CR-01): the Plan completion handler treats a transient envelope-read error as terminal failure.

`handlePlannerJobCompletion` (plan_controller.go:491-505) sets `Status.Phase="Failed"` + `ConditionFailed{Reason: EnvelopeReadFailed}` whenever `EnvReader.ReadOut` errors — a transient PVC/read error wedges the Plan permanently. The milestone (:529-545) and phase (:462-476) handlers treat the same error NON-FATALLY (Pitfall 1: log + defer to children-based succession via `envReadOK`/`envReaderPresent` sentinels). This plan brings the Plan handler up to that pattern.

Purpose: A transient read error is a requeue concern, not terminal state — the Plan should defer to its all-children-Succeeded signal rather than wedge.
Output: Non-fatal read path in the Plan handler + regression spec.

NOTE: This plan edits plan_controller.go / plan_controller_test.go, which Plan 17-01 also touches — hence `depends_on: [17-01]` and wave 2, to avoid a merge collision on the shared file.
</objective>
