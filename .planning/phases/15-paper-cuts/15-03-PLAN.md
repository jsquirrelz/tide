    1. Input hardening (V5): after parseArtifactRef, reject paths containing "..", a leading "/", or shell-metacharacters (quotes, backticks, "$", ";", "&", "|", whitespace) — the path is consumed by a shell command inside the pod. Pass the path to the container via a Pod env var (e.g. ARTIFACT_PATH) and reference it in the command as "$ARTIFACT_PATH" rather than fmt.Sprintf-interpolating it into the sh -c string (defense in depth against injection, T-15-08).
    2. Resolve the Project CR via a controller-runtime Get (the cmd/tide package already constructs a client for other subcommands — reuse that construction) to obtain the Project UID. The PVC subPath is the project UID, NOT the project name (RESEARCH Pitfall 4).
    3. Build the inspector Pod: image busybox:1.36, RestartPolicy Never, volume from PVC claim tide-projects (flag --pvc, default "tide-projects"), volumeMount at /workspace with subPath = project UID, generated name like tide-inspect-<suffix>. Container command implements the D-11 race-free readiness wait IN the pod: a sh -c loop that (a) waits until the file exists, then (b) polls until two consecutive size samples 2 seconds apart are equal (busybox `stat -c %s`), then (c) cats the file once. Existence alone is not enough — the stability window guards against reading a half-written artifact when the writer does not write-then-rename. Document the 2s stability window in a comment.
    4. Create the pod via clientset CoreV1().Pods(ns).Create; defer Delete with context.Background() so cleanup fires on EVERY exit path (T-15-09), but only after the stream has fully drained on the success path.
    5. Bound the whole operation with a --timeout flag on newArtifactGetCmd (default 5m, per RESEARCH Open Question 1): context.WithTimeout around create/wait/stream. Wait for the pod to leave Pending (Running or Succeeded), then open the log stream with Follow:true and io.Copy to stdout — stdout receives ONLY raw artifact bytes (D-10). All progress/status lines ("creating inspector pod...", "waiting for artifact...") go to stderr. On timeout expiry, return a plain non-zero error stating the artifact was not available within the window (D-12) — after the deferred pod delete runs.
    6. Extract the pod-runner as a function var seam (e.g. var inspectorPodRunner = defaultInspectorPodRunner) mirroring tail.go's tailStreamer pattern so tests inject a fake without a live apiserver.
    7. Remove artifactGetDryRun as the default behavior. If artifact_get.go exposes a --dry-run flag today, keep dry-run reachable only behind that explicit flag; if dry-run was unconditional, delete the function. Update the cobra Long help to note required RBAC: pods create/get/delete + pods/log get in the target namespace (the CLI runs under the operator's kubeconfig; enforcement is cluster-side).
