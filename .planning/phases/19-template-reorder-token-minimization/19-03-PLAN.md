<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| rendered prompt → subagent LLM | task_executor text is the only guardrail steering a dispatched coding agent that writes files in a worktree and commits. |
| DeclaredOutputPaths directive → in-pod harness | The template tells the model writes outside DeclaredOutputPaths are rejected; weakening this invites the model to attempt out-of-scope writes (harness still rejects them, but the model wastes iterations / may misbehave). |
| no-push directive → git credential boundary | The template states subagent pods have no git credentials and must not push. This is informational-integrity, not a credential control — the actual control is the absence of credentials in the pod. |
| template change → eval gate | Structural-only gate cannot prove a trimmed executor instruction still steers the model correctly. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-19-06 | Tampering (silent guardrail weakening) | Structural-only eval gate blind to a semantically-weakened-but-parseable executor prompt | mitigate (residual: MEDIUM) | Same residual class as plans 19-01/02. Mitigated by the D-05 human-review checkpoint (plan 19-04) + `make eval`. LLM-judge deferred (EVAL-F1); residual accepted phase-level. |
| T-19-SC | Tampering | npm/pip/cargo installs | accept | No package installs in Phase 19 (RESEARCH §Package Legitimacy Audit). No supply-chain surface. |
