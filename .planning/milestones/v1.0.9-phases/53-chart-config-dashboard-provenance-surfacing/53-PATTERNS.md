```go
func ParsePricingOverrides(s string) (map[string]PriceOverride, error) {
	s = strings.TrimSpace(s)
	if s == "" || s == "{}" {
		return map[string]PriceOverride{}, nil
	}
	var raw map[string]PriceOverride
	if err := json.Unmarshal([]byte(s), &raw); err != nil {
		return nil, fmt.Errorf("pricing overrides: invalid JSON: %w", err)
	}
	for modelID, p := range raw {
		if p.InputCentsPerMTok <= 0 {
			return nil, fmt.Errorf(
				"pricing overrides: model %q has invalid inputCentsPerMTok %d (must be > 0)",
				modelID, p.InputCentsPerMTok)
		}
		// ... more field validation, same shape ...
	}
	return raw, nil
}
```
`ParseVerifyLevelDefaults(s string) (map[string]LevelVerifyDefault, error)` mirrors this exactly: empty/`"{}"` → empty map, nil error; `json.Unmarshal` with a wrapped error; a validation loop over the 5 level keys rejecting negative `maxIterations` or an unrecognized `onExhaustion` enum value, with the level name in the error string (mirrors `modelID` in the error string above — "the operator's fix is mechanical" per the doc comment). ## Shared Patterns

### Chart-generation discipline (blocks EVERY chart-touching file this phase)
**Source:** `Makefile:806-816` (`helm-controller` target), `hack/helm/augment-tide-chart.sh` header, CI's `helm-lint` job.
**Apply to:** `hack/helm/tide-values.yaml`, `hack/helm/augment-tide-chart.sh`, `hack/helm/verify-posture-configmap.yaml`.
`charts/tide/values.yaml` and `charts/tide/templates/deployment.yaml` are GENERATED — hand-editing them passes local review and fails CI's `helm-lint`/`verify-chart-reproducible` diff check. Every value/env/arg addition in this phase is authored in `hack/helm/`, then `make helm-controller` is run and the regenerated `charts/tide/` output is committed in the SAME commit as the `hack/helm/` source change (RESEARCH Pitfall 1, D-10's own naming of this as the v1.0.8-lesson gate to run in-phase).

### Fail-closed startup validation (ASVS V5)
**Source:** `pkg/dispatch/pricing.go` `ParsePricingOverrides` + `cmd/manager/main.go:244-249`.
**Apply to:** `pkg/dispatch/verify_defaults.go`'s `ParseVerifyLevelDefaults`, and its call site in `main.go`.

### Empty-image-ref benign-skip (dispatch-site defense)
**Source:** `internal/controller/task_controller.go:2178-2182` (`dispatchVerifier`'s `VerifierImage == ""` guard), doc comment cross-references `TIDE_REPORTER_IMAGE`/`TIDE_PUSH_IMAGE` skips.
**Apply to:** any new verifier-dispatch call site touched by D-04's enablement gate.
When a required image ref is empty, log at `Info` level and leave the CRD benignly parked rather than building a Job with an empty image (unschedulable, silent). Same idiom applies to a disabled level: no dispatch, no error, level proceeds through the "off" path.

### Closed allowlist for user-influenced path/kind params (ASVS V4)
**Source:** `cmd/dashboard/api/artifacts.go:51-60` (`artifactKinds`).
**Apply to:** the `"task"` addition to this same map.
Never loosen a closed-map allowlist to a regex or open string, even when adding a new legitimate value — extend the map with one more literal key.

### Additive, defensively-coerced vocabulary tables
**Source:** `dashboard/web/src/components/StatusBadge.tsx` (`STATUS_TABLE`) + `ConditionBadge.tsx` (`CONDITION_TABLE`), both already designed for unknown-value fallback (`coerceStatus`'s unknown→`"Pending"`, `ConditionBadge`'s unknown→`null`).
**Apply to:** the `Verifying`/`VerifyHalted`/`VerifyHalt` row additions.
New states are ADDITIVE ROWS on an existing table, never a bespoke new component. Server and frontend table can land in either commit order without breaking rendering (unknown types degrade gracefully on both sides).

### `--color-status-blocked` = policy-halted, operator-recoverable (never the error color)
**Source:** `BudgetBlocked`/`BillingHalt`/`PushLeakBlocked` rows (both `StatusBadge.tsx` and `ConditionBadge.tsx`).
**Apply to:** `VerifyHalt`/`VerifyHalted`.
This is what makes "distinct from Failed" true by construction (D-09) — `Failed` uses `--color-status-error`; every halt-class state (budget, billing, verify) shares the blocked-family color, distinguished from each other by icon + label only, never by falling back to the error palette.

### Section-separator convention in detail drawers
**Source:** `TaskDetailDrawer.tsx`'s repeated `border-t border-[var(--color-border-subtle)] px-6 py-4` + `data-testid="drawer-*"` shape (metadata grid, Phoenix link, actions row, conditions list).
**Apply to:** the new Loop provenance section.
