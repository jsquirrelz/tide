// TaskReconcilerDeps carries the dispatch-related dependencies for TaskReconciler.
// Mirrors HelmProviderDefaults precedent at dispatch_helpers.go:60-69. Fields

    SigningKey     []byte

    CredproxyImage string

    client.Client

    MaxConcurrentReconciles int

<additional_context>
**Why this is separate from P3.1:** the P3.1 extraction touches the dispatch BODY (extracting helper methods); P3.2 touches the dispatch SURFACE (the struct shape). They affect different test types (P3.1 needs per-method tests; P3.2 needs struct-literal-shape updates everywhere). Combining would inflate one commit's diff to ~15-25 file changes — keeping them separate keeps `git blame` readable.

**Forward compatibility from Plan 04.1-02:** the wiring_test.go file created in Plan 02 documented this transition in its NOTE block. Plan 08 honors that note by updating the Task subtest case.

**Pool fields stay direct:** `PlannerPool` and `ExecutorPool` are NOT moved into Deps per RESEARCH.md §P3.2 §Known pitfalls — they're a different conceptual category (concurrency limiters, not dispatch-tier deps). Leaving them direct also avoids a needlessly large blast radius into the pool tests.

**Value field, not pointer:** `Deps TaskReconcilerDeps` (value) not `Deps *TaskReconcilerDeps` (pointer). Copying ~8 pointer/value fields at construction time is cheaper than dereferencing through a pointer at every dispatch. RESEARCH.md §P3.2 explicit.
