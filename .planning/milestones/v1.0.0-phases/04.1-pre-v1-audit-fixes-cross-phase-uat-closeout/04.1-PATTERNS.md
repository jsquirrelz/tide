## Orienting note for the planner

Phase 04.1 is overwhelmingly about **co-locating** new code with its production analog, not inventing patterns. The list below pairs each touched file with the single closest existing file that:

1. Carries the same role (controller, dispatch helper, schema, webhook, CI workflow, …).
2. Lives next to the code being changed (so `git blame` stays coherent and `// Phase 04.1 …` markers extend the existing comment convention).
3. Has the patterns that should be **literally copied** (import block, comment style, error wrapping, condition-set vocabulary, MergeFrom+Patch idiom, fake-client builder).

The two items with real pattern surface area are **P1.2 (planner Job contract extraction)** and **P4.2 (Project schema field + sidecar env var + webhook validation)**. Everything else is one-to-five-line surgical mods.
