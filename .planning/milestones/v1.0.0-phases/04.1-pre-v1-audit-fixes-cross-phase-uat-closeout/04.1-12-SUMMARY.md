## Lessons (PIT-7 framing — iteration, not closeout-on-first-try)

This plan ran 5 iterations. Each iter-N's BLOCKED gate surfaced new cascades; per CLAUDE.md PIT-7, the user explicitly directed surface-and-ask after iter-2 and again after iter-3, iter-4. Each AskUserQuestion checkpoint preserved the iteration loop while ensuring user agency.

**Phase 02.2 echo:** the cascade pattern (12 iterations to close one timing fix) repeated here at smaller scale (5 iterations for one Layer B UAT closeout). The CLAUDE.md PIT-7 instruction is load-bearing — without it, an autonomous loop would have continued chasing fixes past iter-3 without surfacing to the user that the cascade shape was changing.

**Key insights:**

1. **The 7m → 12m → 15m → 18m timeout creep was diagnostic of growing suite depth.** Each iter exposed more specs running successfully (specs that previously skipped via ctx-expiry now actually run), which surfaces more failure shapes. The fix-then-rerun loop accumulated coverage iteratively.
2. **Cascade 4-a (fixture label backfill) was the highest-leverage fix** — it closed THREE distinct test failures (output_test, caps_test, failure_test) with a single 3-file edit. Phase 04.1 P1.4 was the upstream change that broke these fixtures; the cascade walked from "test fails" → "P1.4 commit" → "missing label" in 3 plans.
3. **Cascade 4-b (helm-failure-resilient secret mirror)** revealed a structural test-infra coupling: `applyController()` returned early on helm `--wait` timeout, silently skipping post-helm setup steps. Extracting `ensureSigningKeySecret` to BeforeSuite-after-waitForControllerReady decoupled it from the helm-success contract.
4. **Phase 03 cascades (push_lease, chaos_resume line 230, Plan-pod credproxy) are real but out of scope.** Scope-defer was the right call per the user's iter-5 decision; they belong to a Phase 03 follow-up plan.
5. **The Layer A envtest flake (TestGateApproveFlow)** is pre-existing intermittent — confirmed by isolated re-runs (Run A pass, Run B fail). Not regressed by my changes.
