# Phase 04.1 Plan 06: CI / Tooling Hygiene Bundle (P2.1-P2.4) Summary

**One-liner:** Wired dagimports.Analyzer into tide-lint multichecker, added two named verify-* CI steps, guarded go-mod-tidy with git-diff exit-code checks, and committed a workflow_dispatch-only live-e2e workflow.

## What Was Built

### P2.1 — dagimports.Analyzer registered in cmd/tide-lint/main.go

Added `dagimports.Analyzer` as the 4th entry in `multichecker.Main(...)`. The analyzer is at `tools/analyzers/dagimports/` and was already well-formed with testdata and `analyzer_test.go`. This wires the DAG-05 constraint as a redundant IDE-time fast-feedback gate alongside the load-bearing `make verify-dag-imports` shell target (which uses `go list -deps` for transitive coverage; the analyzer covers direct imports only).

**Files modified:** `cmd/tide-lint/main.go`

### P2.2 — Two named verify-* steps in .github/workflows/ci.yaml

Inserted after "Verify no blocking I/O in reconcilers (Pitfall 1)" and before "Run custom analyzers":

- `Verify pkg/dispatch imports (SUB-01)` — runs `make verify-dispatch-imports`
- `Verify import firewall (SUB-05 — alias for tide-lint providerfirewall)` — runs `make verify-import-firewall`

Both Makefile targets already existed (lines 409, 444). Making them named CI steps gives explicit badge coverage over implicit coverage via `make tide-lint`.

**Files modified:** `.github/workflows/ci.yaml`

### P2.3 — go-mod-tidy diff-check in test.yml + test-e2e.yml

After `go mod tidy`, added `git diff --exit-code go.mod go.sum` in both workflows. Module drift no longer absorbs silently into a CI run. Existing step names preserved exactly:
- `test.yml`: step name `Running Tests` preserved
- `test-e2e.yml`: step name `Running Test e2e` preserved

**Files modified:** `.github/workflows/test.yml`, `.github/workflows/test-e2e.yml`

### P2.4 — .github/workflows/live-e2e.yml (new) + docs/live-e2e.md updated

**New file:** `.github/workflows/live-e2e.yml` — `workflow_dispatch:` ONLY per locked user decision 2. No `schedule:` cron committed at v1.0. Uses SHA-pinned action references copied from `test-e2e.yml` (`actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd` and `actions/setup-go@4b73464bb391d4059bd26b0524d20df3927bd417`). Runs `make test-e2e-live` with `ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}`.

**Updated:** `docs/live-e2e.md` — replaced the embedded template workflow block (lines 43-81 of original) with a "## CI integration" section pointing at the committed `live-e2e.yml`. The template had a `schedule:` cron that contradicts locked decision 2; replaced with instructions for operators who want nightly runs in their fork. All other doc sections (build tag gates, budget rationale, cost baseline, fixture pinning, troubleshooting) preserved intact.

**Files:** `.github/workflows/live-e2e.yml` (created), `docs/live-e2e.md` (modified)
