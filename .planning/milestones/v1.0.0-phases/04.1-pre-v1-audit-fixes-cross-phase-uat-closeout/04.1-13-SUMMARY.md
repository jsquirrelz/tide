## Lessons Learned (PIT-7 framing)

- **Stale-UAT detection is high-leverage.** Per locked decision 4, the researcher correctly flagged items 3 + 4 as already-closed in Phase 4 source. Re-verification (1 envtest focus run, 14s wall time) vs. authoring two new fix plans (likely 2–4 hours each) — the framing-as-iteration discipline saved ~6 hours of churn.
- **Evidence reuse > redundant test runs.** The Wave-6 cascade-13 closeout invocation (`make test-int`, 12.9-minute Layer B run) already covered all three Wave-7 Item-1 specs (chaos-resume + push-lease + up-stack-dispatch live in the same suite). Re-running would have cost 13 minutes of wall time for zero additional information. The orchestrator's explicit "do NOT re-run" instruction in the executor prompt is the cleanest way to enforce this — symbol-path evidence + suite-summary line is sufficient.
- **Cost-gating as architecture, not workflow.** Locked decision 6 says the $1.00 cap in `live-claude-project.yaml:76` IS the enforcement — no per-execution re-prompt, no human-in-the-loop on every dispatch. The cap is the trust boundary, encoded once in the CRD spec. Workflow-side gates (BeforeSuite skip + build tag + Makefile target) layer on top but don't replace the cap-as-architecture decision. When the API key is absent, the defer path is structural, not a failure mode.
- **Line-number drift between RESEARCH and source happens.** RESEARCH.md cited `milestone_controller.go:319` / `phase_controller.go:271` / `registry.go:118-124` / `project_controller.go:484-510`; actual numbers post-Phase 4 cascades are `:419` / `:340` / `:73,128,130,157` / `:527`. Symbol-based grep (per Plan-CHECK Flag 4 defensive targeting) survived the drift cleanly — line-number-only assertions would have false-failed. The symbol-targeting discipline in the plan's `must_haves.key_links.pattern` field is the right one.
