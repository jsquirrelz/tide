NOTE on http:// vs https:// in the new CEL rule: The new rule explicitly accepts BOTH http:// AND https:// (two separate startsWith checks). This is intentional — the in-cluster medium sample uses http:// (anonymous git-http server); production uses https://. Both must be accepted. The old rule used startsWith('http') which matched both but also matched file:// (startsWith('http') is false for 'file://' but the rule had a third OR clause for file://). The new rule is explicit and precise.
