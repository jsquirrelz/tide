Contract from pkg/dispatch (Phase 2, UNCHANGED):
```
type Subagent interface {
    Run(ctx context.Context, in EnvelopeIn) (EnvelopeOut, error)
}
```

Phase 3 anthropic Subagent shape (this plan ships):
- Package: `internal/subagent/anthropic`
- Public type: `type Anthropic struct { ... }` and `func New(opts Options) *Anthropic`
- Method: `func (a *Anthropic) Run(ctx context.Context, in pkgdispatch.EnvelopeIn) (pkgdispatch.EnvelopeOut, error)`

Claude Code CLI invocation (RESEARCH.md verified):
```
claude -p <prompt> --model <model> --bare --output-format stream-json --verbose --include-partial-messages
```
The `--bare` flag (RESEARCH §"Critical new finding") skips auto-discovery of project files; required for in-Pod operation where the working dir is the per-Task worktree (Phase 3 D-B4).

Env vars to set on the exec'd claude process:
- `ANTHROPIC_BASE_URL=<in.ProxyEndpoint>` (Phase 2 D-C1 credproxy sidecar)
- `ANTHROPIC_API_KEY=<in.SignedToken>` (HMAC-signed token, NEVER raw API key — Phase 2 D-C1..C4)
- `NODE_EXTRA_CA_CERTS=/etc/tide/proxy/ca.crt` (credproxy self-signed cert)
- Inherit existing env; do NOT mount ~/.claude/ (CLAUDE.md anti-pattern).

Stream-json event shape (RESEARCH.md verified, stable v2.1.139..v2.1.142):
```
{"type":"system/init", ...}
{"type":"stream_event", "delta":{"type":"text_delta", "text":"..."}}
{"type":"result", "result":"final assistant text", "usage":{"input_tokens":N, "output_tokens":N, "cache_read_input_tokens":N, "cache_creation_input_tokens":N}, "total_cost_usd":F}
```

Top-level keys are snake_case. Map to pkg/dispatch.Usage's camelCase fields per D-C5.

Prompt templates (compiled-in Go templates per CLAUDE.md anti-pattern "Don't vendor GSD Markdown"):
- `templates/milestone_planner.tmpl` — planner prompt for authoring MILESTONE.md (level=milestone, role=planner).
- `templates/phase_planner.tmpl` — planner prompt for phase brief (level=phase, role=planner).
- `templates/plan_planner.tmpl` — planner prompt for PLAN.md (level=plan, role=planner).
- `templates/task_executor.tmpl` — executor prompt (level=task, role=executor).

Template content: minimal v1 prompts referencing README.md spec; the executor is the one to draft them (per CONTEXT.md "Claude's Discretion: Prompt-template content"). Keep them as Go `text/template` files; pass the EnvelopeIn struct as the template context so the template can reference `{{.Provider.Model}}`, `{{.Level}}`, `{{.TaskUID}}`, etc.
