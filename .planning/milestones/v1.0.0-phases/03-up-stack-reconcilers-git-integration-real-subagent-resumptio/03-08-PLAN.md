Phase 2 reconcileDispatch 12-step shape to mirror (task_controller.go:192-425):
1. Terminal short-circuit (Phase=Succeeded|Failed return).
2. On Phase=Running: check Job terminal state.
3. Resolve owning Project (walk ref chain).
4. Acquire executorPool (Phase 3 up-stack switches to plannerPool).
5. Resolve provider (Phase 3 new step via dispatch_helpers.ResolveProvider).
6. Build EnvelopeIn (Phase 3 new function BuildPlannerEnvelope for up-stack).
7. Job creation with deterministic name.
8. Status patch (Phase=Running + AuthoringPlanner condition).
9. handleJobCompletion: read EnvelopeOut, materialize child CRDs, fire next-level work.

Shared dispatch helper API (this plan creates):

```

// BuildPlannerEnvelope: planner-side analog of task_controller.go buildEnvelopeIn.
// Returns the EnvelopeIn struct + the JSON bytes ready to write to PVC.
func BuildPlannerEnvelope(level string, parent metav1.Object, project *tideprojectv1alpha1.Project, attempt int, token string, caps pkgdispatch.Caps, proxyEndpoint string) (pkgdispatch.EnvelopeIn, []byte, error)

// MaterializeChildCRDs server-side-creates child CRDs from EnvelopeOut.ChildCRDs.
// Enforces Kind allowlist (T-308 mitigation) and sets OwnerRef.
func MaterializeChildCRDs(ctx context.Context, c client.Client, scheme *runtime.Scheme, parent metav1.Object, parentKind schema.GroupVersionKind, children []pkgdispatch.ChildCRDSpec) error
```

Kind allowlist (T-308 mitigation): MaterializeChildCRDs MUST reject any child whose Kind is not in `{Milestone, Phase, Plan, Task, Wave}` (these are the only tideproject.k8s Kinds). Reject with structured error; calling reconciler patches Status.phase=Failed.

ProjectReconciler.Reconcile extensions (existing + new):
- Existing: init Job dispatch (ensureInitJob lines 207-219).
- New (Phase 3): clone Job dispatch right after init Job succeeds; branch-name init right before clone (Status.git.branchName = `tide/run-<project.Name>-<unix>`).
- New: push Job dispatch when a level boundary completes (D-B2 — Plan all-Tasks-Succeeded, Phase all-Plans-done, Milestone all-Phases-done, Project=Complete). The decision to fire push lives on whichever reconciler observes the level-completion event; ProjectReconciler is the natural orchestrator (it watches all child Kinds via Owns()).
- New: push Job completion handler → patches Status.git.lastPushedSHA from push-result envelope; on lease-rejection → Status.phase=PushLeaseFailed.
- New: bypass-annotation watcher — predicate matches `tideproject.k8s/bypass-push-lease=true`; clears Status.phase=PushLeaseFailed and requeues.

SetupWithManager extensions (per reconciler):
- MilestoneReconciler: Owns(&batchv1.Job{}) + Owns(&Phase{}) + Owns(&Job{}) — child-status drives parent reconcile.
- PhaseReconciler: Owns(&Job{}) + Owns(&Plan{}).
- PlanReconciler: Owns(&Job{}) + Owns(&Task{}) + existing Wave watch.
- ProjectReconciler: Owns(&Job{}) + Owns(&Milestone{}) + new push-job bypass-annotation predicate.

Deterministic Job names per level (mirrors Phase 2 D-B5):
- Milestone planner Job: `tide-milestone-<milestone-uid>-<attempt>` (where attempt = Milestone.Status.Attempt+1).
- Phase planner Job: `tide-phase-<phase-uid>-<attempt>`.
- Plan planner Job: `tide-plan-<plan-uid>-<attempt>`.
- Clone Job: `tide-clone-<project-uid>` (one-shot; no attempt suffix needed).
- Push Job: `tide-push-<project-uid>` (D-B5 serialization key; no level suffix).
