**Cascade-10 (PVC namespace-scoping) — fix-shape pivot from 02.2-11-SUMMARY's Option A:**

Plan 02.2-11's SUMMARY proposed Option A: per-namespace `tide-projects` PVC declarations in every test fixture YAML. After implementation observation, this plan adopts a more architecturally clean approach:

- The manager-side `EnvelopeReader` is reimplemented as `PodStatusEnvelopeReader`, which reads `EnvelopeOut` from the subagent container's terminationMessagePath via `Pod.status.containerStatuses[].state.terminated.message`. This removes the manager's dependency on mounting the namespace-local Task PVC entirely.
- The PVC remains Pod-local (a fixture-level concern) and is mirrored into test namespaces via a new `ensureProjectsPVC(ns)` helper (analogous to the existing `ensureSubagentSA(ns)`). The wave-test fixture also declares the PVC inline so `kubectl apply -f testdata/three-task-wave.yaml` is self-contained.
- `EnvelopeReader.ReadOut(ctx, taskUID)` becomes `ReadOut(ctx, projectUID, taskUID)`; `FilesystemEnvelopeReader` (now the fallback for in-process/local tests) joins `WorkspaceRoot/{projectUID}/workspace/envelopes/{taskUID}/out.json` — matching the production workspace layout.
- The stub-subagent gains a `writeTerminationMessage(data)` helper that writes the envelope JSON to `/dev/termination-log` (overridable via `TIDE_TERMINATION_MESSAGE_PATH` for tests) so the manager-side Pod-status reader has something to read.

**Cascade-11 (Secret namespace-scoping) — direct fix-shape per 02.2-12-VERIFICATION.md's classification:**

Same shape as cascade-10's PVC mirroring: helm creates `tide-signing-key` only in `tide-system`, but credproxy sidecars `envFrom: secretRef: name: tide-signing-key` in the Pod's own namespace. New `ensureSigningKeySecret(ns)` helper reads `data.TIDE_SIGNING_KEY` from `tide-system/tide-signing-key` (via `kubectl get secret -o jsonpath`) after `helm upgrade --install` completes, then applies a base64-data-identical Secret in the target namespace. `createNamespace(ns)` now calls `ensureSigningKeySecret(ns)` alongside `ensureSubagentSA(ns)` and `ensureProjectsPVC(ns)`.
