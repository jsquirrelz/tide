**Cascade-9 (THIS PLAN — newly surfaced with two sub-classes):**

**Sub-class A — job.spec.activeDeadlineSeconds=60 too tight for credproxy initContainer under kind:**
The `PodJobBackend.BuildJobSpec` sets `activeDeadlineSeconds` from the Task's configured budget. For
the wave-test fixture tasks (alpha, beta), the budget maps to 60s. Under kind, the credproxy
initContainer (`tide-credproxy`) takes longer than 60s to complete:
- envelope-writer initContainer runs first
- tide-credproxy initContainer starts, presumably does certificate setup / proxy handshake
- The 60s deadline fires before either initContainer completes
- Pod is killed; Job reports DeadlineExceeded

Fix shape for cascade-9A: either (a) increase the wave-test fixture tasks' budget/deadline to give
credproxy initContainer sufficient time under kind (e.g., 120s or 240s), OR (b) restructure the
credproxy initContainer to complete faster (less relevant for test budget fix), OR (c) make the
job's `activeDeadlineSeconds` configurable per test vs production profile.

**Sub-class B — Layer A envtest SUB-02 Wave roll-up timeout (spec-flake):**
The "Wave roll-up: Succeeded when all Tasks Succeeded" spec at `indegree_test.go:232` fails with a
20s timeout on `Wave.Status.Phase == "Succeeded"`. This is a spec-flake caused by the newly active
`reconcileDispatch` in envtest: Tasks in the indegree test namespace have no parent Project, so
`resolveProject` at Step 3 fails with an error → TaskReconciler returns error → requeue → repeated
failed reconciles create work-queue load that delays the WaveReconciler's observation of the
test-patched Task.Status.Phase=Succeeded. The spec passes when run in isolation (label filter SUB-02).
Fix shape: increase the wave roll-up spec's `Eventually` timeout from 20s to 30-45s, OR add
proper Project+Plan+Task hierarchy in the indegree test fixture so reconcileDispatch doesn't error.

**Sub-class C — budget-vs-actual (wall time 1486s >> 600s):**
With a real dispatch path running, the kind integration suite's total execution time exceeds the
600s Makefile budget. Contributors: 
- kind cluster creation + image loading + BeforeSuite helm install: ~250s baseline
- Layer A envtest with 20s SUB-02 timeout: ~77s
- AC1 wave spec running real Jobs that fail at 60s deadline, then Ginkgo 5m timeout fires: ~300s
- Remaining overhead
Fix shape: split Layer A and Layer B into separate Makefile targets with separate budgets, OR reduce
BeforeSuite overhead, OR fix the job activeDeadlineSeconds to allow specs to complete faster.

**Cascade-9 ValidationState gap (deferred — same as debug session §"Secondary risk"):**
`.planning/debug/credproxy-backoff-suppression.md` §"Secondary risk" identified that `PlanReconciler.
reconcileWaveMaterialization` short-circuits at `plan_controller.go:147` if `plan.Status.ValidationState
!= "Validated"`. The Plan admission webhook (`internal/webhook/v1alpha1/plan_webhook.go`) validates

wave-test AC1 failure if WaveReconciler depends on Wave objects created by PlanReconciler.
reconcileWaveMaterialization. This warrants investigation in a follow-up debug session.
