## Leader-Election Exception (only "wildcard-tolerant" ClusterRole in the system)

The leader-election `Role` at `config/rbac/leader_election_role.yaml` is namespace-scoped (Role, not ClusterRole — it operates against the operator's own install namespace's coordination.k8s.io/leases) and declares ENUMERATED verbs (`get;list;watch;create;update;patch;delete`) — no wildcards. The cluster-scoped ClusterRoles in `config/rbac/` are:

| ClusterRole | Source | Purpose | Wildcards? |
| --- | --- | --- | --- |
| `manager-role` (config/rbac/role.yaml) | controller-gen from kubebuilder markers | The orchestrator's own permissions | ZERO (this plan's contract) |
| `metrics-auth-role` | kubebuilder scaffold | Authn/authz on the /metrics endpoint | NO wildcards |
| `metrics-reader` | kubebuilder scaffold | Read /metrics (Prometheus scrape) | NO wildcards |
| `<kind>-admin-role` × 6 | kubebuilder scaffold | Allow cluster admin to delegate full mgmt to users | `verbs: ['*']` (multi-line YAML — explicitly NOT used by the operator itself, documented as "This rule is not used by the project tide itself") |
| `<kind>-editor-role` × 6 | kubebuilder scaffold | Allow delegation of CRUD to users | enumerated verbs (no wildcards) |
| `<kind>-viewer-role` × 6 | kubebuilder scaffold | Allow delegation of read to users | enumerated verbs (no wildcards) |

The admin roles' `verbs: ['*']` literal is multi-line:
```yaml
  verbs:
  - '*'
```
which the plan's same-line regex `'verbs:.*"?\*"?'` does NOT match by design. The contract enforced by `make verify-no-rbac-wildcards` is: the operator's own manager-role (and any hand-edit to existing same-line wildcard YAMLs) is wildcard-free. The kubebuilder-scaffolded admin roles are documented escape hatches for cluster admins to delegate to end users; they are not part of the orchestrator's effective permissions.

## Two New Makefile Targets

```makefile
##@ RBAC wildcard gates (AUTH-03 / Pitfall 15)

.PHONY: verify-no-rbac-wildcards
verify-no-rbac-wildcards: ## Assert no RBAC wildcards in config/rbac/ generated manifests.
	@echo "verifying no RBAC wildcards in config/rbac/ (AUTH-03 / Pitfall 15)..."
	@MATCHES=$$(grep -nrE 'verbs:.*"?\*"?|resources:.*"?\*"?' config/rbac/ 2>/dev/null || true); \
	if [ -n "$$MATCHES" ]; then \
		echo "AUTH-03 violation: RBAC wildcards detected:"; \
		echo "$$MATCHES"; \
		exit 1; \
	fi
	@echo "OK: no RBAC wildcards"

.PHONY: verify-rbac-marker-discipline
verify-rbac-marker-discipline: ## Assert no wildcard kubebuilder:rbac markers in source.
	@echo "verifying no RBAC wildcards in source markers (AUTH-03 / Pitfall 15)..."
	@# Scoped to *_controller.go (production reconciler files only). Test files
	@# legitimately contain marker-shaped string literals as fixtures
	@# (e.g. internal/controller/rbac_guard_test.go's TestRBACMarkerDiscipline*).
	@MATCHES=$$(grep -nE 'kubebuilder:rbac.*verbs=\*|kubebuilder:rbac.*resources=\*' internal/controller/*_controller.go || true); \
	if [ -n "$$MATCHES" ]; then \
		echo "AUTH-03 violation: RBAC wildcard markers detected:"; \
		echo "$$MATCHES"; \
		exit 1; \
	fi
	@echo "OK: no RBAC wildcard markers"
```

Both exit 0 against the current state. Both are wired into `.github/workflows/ci.yaml` as named steps; future PRs that reintroduce wildcards in either layer fail the gate at PR time.

## rbac_guard_test.go Contract (revision Warning 4)
