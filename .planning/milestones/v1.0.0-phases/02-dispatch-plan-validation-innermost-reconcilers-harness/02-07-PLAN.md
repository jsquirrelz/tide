<objective>
Build the budget + rate-limiter package: an in-process token-bucket store keyed by Secret UID (FAIL-03), Project-Status budget roll-up + cap check (FAIL-04), pre-charge on Manager restart (D-D1), and a single bounded-cardinality Prometheus counter for rate-limit hits observability.

Purpose: The budget package is the controller-side state for two intertwined-but-distinct concerns:
1. **Rate-limit bucket (per-Secret-UID)** — sub-second-granularity gating between the orchestrator and the LLM provider; lives in-memory only (D-D1) because etcd churn at this granularity is unacceptable.
2. **Budget tally (per-Project)** — durable across restarts via `Project.Status.Budget` (D-D2); one Status write per Task completion (low churn).

The two homes are deliberate per CONTEXT "Specifics §"In-memory rate-limit bucket + Project.Status budget tally is the two-cache split"). Don't unify.

Output: A `internal/budget` Go package with five concerns separated into files (bucket, precharge, cap, tally, metrics). All tests are stdlib `testing` (NOT Ginkgo) per PATTERNS.md leaf-helper convention. Plan 09's TaskReconciler dispatch body calls into this package; Plan 12's cmd/manager bootstraps the Store + runs PreCharge at startup.
</objective>

```
type Limits struct {
    RequestsPerMinute int  // 0 means unlimited
    BurstSize         int  // typically 10 by default
}

type Store struct {
    m sync.Map  // map[string /* secretUID */]*rate.Limiter
}

func NewStore() *Store

// ForSecret returns (creates lazily, then caches) the rate.Limiter for secretUID.
// If limits.RequestsPerMinute is 0, returns nil (caller treats as "no rate limit").
func (s *Store) ForSecret(secretUID string, limits Limits) *rate.Limiter

// Evict removes a bucket (called when a Secret is deleted).
func (s *Store) Evict(secretUID string)
```
