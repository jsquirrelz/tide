<objective>
Build the signed-token credential proxy (HARN-03). Two artifacts together: an `internal/credproxy` Go package providing HMAC sign/verify + self-signed cert minting + HTTPS reverse-proxy server, and a `cmd/credproxy` binary entrypoint + Dockerfile. Per D-C1, the credproxy runs as a Kubernetes 1.33 native sidecar (`initContainer` with `restartPolicy: Always`) inside every Task Job — the subagent container talks to it over localhost HTTPS, never seeing the raw `ANTHROPIC_API_KEY`.

Purpose: Pitfall 18 (secret leakage) defense at the env layer. The subagent container's env has `ANTHROPIC_API_KEY=<signed-token>` (bound to the specific Pod's `TIDE_TASK_UID`); the sidecar's env has the real key via `envFrom: [secretRef: <Project.Spec.providerSecretRef>]`. A token leaked outside the Pod is useless (validation fails on `expectedTaskUID` mismatch). RESEARCH.md Pattern: HMAC sign + verify (stdlib `crypto/hmac` + `hmac.Equal` time-constant compare).

Output: A self-contained `internal/credproxy` package + `cmd/credproxy` binary + Dockerfile producing `ghcr.io/jsquirrelz/tide-credproxy:v0.1.0-dev`. Plan 08's PodJobBackend wires this into every Task Job's PodSpec as the sidecar.
</objective>

`internal/credproxy/token.go` API (D-C3):
```
// Token layout (base64.RawURLEncoding-decoded):
//   [0..16)  nonce        — 16 random bytes
//   [16..24) expiry       — int64 unix-seconds (big-endian)
//   [24..56) mac          — sha256.Size = 32 bytes
// Signed payload = (nonce || expiry-bytes || taskUID-bytes).
const (
    NonceLen  = 16
    ExpiryLen = 8
    MacLen    = sha256.Size  // 32
    TokenLen  = NonceLen + ExpiryLen + MacLen  // 56
)

// Sign produces base64.RawURLEncoded token for taskUID, valid until time.Now().Add(validFor).
// Returns error if signingKey is empty or rand fails.
func Sign(signingKey []byte, taskUID string, validFor time.Duration) (string, error)

// Verify returns nil iff MAC valid, not expired, and matches expectedTaskUID.
// Distinct error types: ErrBadTokenLength, ErrExpired, ErrTaskUIDMismatch, ErrBadMAC.
// (Use sentinel errors so callers can errors.Is the specific failure.)
func Verify(signingKey []byte, token string, expectedTaskUID string) error
```

`internal/credproxy/cert.go` API (D-C2):
```
// MintSelfSignedCert generates ECDSA P-256 key + self-signed cert with
// CN=tide-credproxy, SANs DNSNames:[localhost], IPAddresses:[127.0.0.1, ::1].
// Writes cert.pem, ca.crt (identical content), key.pem to dir.
// Validity per validity arg; recommend 24*time.Hour for Job-lifetime pods.
func MintSelfSignedCert(dir string, validity time.Duration) error
```

`internal/credproxy/server.go` API:
```
type Proxy struct {
    SigningKey      []byte    // from Secret tide-signing-key
    ExpectedTaskUID string    // from env TIDE_TASK_UID
    UpstreamBaseURL string    // "https://api.anthropic.com"
    RealAPIKey      string    // from envFrom: secretRef <providerSecretRef>
    ListenAddr      string    // "127.0.0.1:8443"
    CertFile        string    // /etc/tide/proxy/cert.pem
    KeyFile         string    // /etc/tide/proxy/key.pem
}

func (p *Proxy) Handler() http.Handler
func (p *Proxy) ListenAndServe(ctx context.Context) error
```

Behavior of `Handler()`:
1. Extract bearer token from `Authorization` header (strip "Bearer " prefix) OR fall back to `x-api-key` header.
2. Call `Verify(p.SigningKey, token, p.ExpectedTaskUID)`.
3. On Verify failure: respond 401 with body `unauthorized: <err.Error()>`.
4. On Verify success: rewrite request headers via `httputil.ReverseProxy.Director`:
   - Set `Authorization: Bearer <RealAPIKey>` (Anthropic SDK header form)
   - Set `x-api-key: <RealAPIKey>` (Anthropic SDK alternate header form)
   - Strip any incoming `Authorization` (replaced) and pass through `anthropic-version` and other headers untouched.
   - Set `req.Host = upstream.Host`
5. Forward via `httputil.NewSingleHostReverseProxy(upstream)`.

`cmd/credproxy/main.go` (D-C2, RESEARCH.md "cmd/credproxy/main.go" pattern):
- Flag parsing: `--listen-addr` (default `127.0.0.1:8443`), `--cert-dir` (default `/etc/tide/proxy`), `--upstream-url` (default `https://api.anthropic.com`), `--cert-validity` (default `24h`).
- Env reads: `TIDE_TASK_UID` (required), `TIDE_SIGNING_KEY` (required; loaded from `Secret tide-signing-key` envFrom), `ANTHROPIC_API_KEY` (required; loaded from Project's providerSecretRef envFrom).
- Startup order: (a) parse flags + env, fail-fast on missing required, (b) `MintSelfSignedCert(certDir, certValidity)`, (c) construct Proxy, (d) `srv.ListenAndServe(ctx)` under `signal.NotifyContext(...)` so SIGTERM cancels cleanly.
- Use `zap`-behind-logr per Phase 1 logger convention (same pattern as `cmd/manager/main.go`).
