## Decisions Made

### D-01 — Credproxy boot-banner approach (LOCKED)
Plan 03 will implement: `fmt.Fprintf(os.Stdout, "credproxy listening on %s\n", listenAddr)` inserted after `log.Info("starting credential proxy", ...)` in `cmd/credproxy/main.go`. The structured zap log stays; the banner is the operator-readable/test-grep-readable signal. Rejected (b) updating the test grep to match zap JSON because boot banners are a common K8s-operator convention worth keeping.

### D-02 — Per-test subagent-SA strategy (LOCKED)
Plan 02 will implement: a new `ensureSubagentSA(ns string)` helper in `test/integration/kind/suite_test.go` (structural analog of the existing `applyYAML` helper at lines 385-394) that creates a zero-RBAC `tide-subagent` ServiceAccount in any per-test namespace. Rejected cluster-scoped SA (impossible — K8s SAs are namespace-scoped) and rejected a chart post-render hook (out of scope; tests own their per-namespace setup).

### D-03 — TIDE_REQUIRE_CONTROLLER default for Phase 02.1 (LOCKED)
Keep the existing code default of `unset` (Phase 2 D-H3 contract: silent skip when controller not ready). Plan 04 wires `TIDE_REQUIRE_CONTROLLER=1` in the CI workflow's `env:` block to make install failures fail CI loudly. Plan 02 does NOT invert the code default. Local dev iterations may unset for partial-coverage debugging.

### Hypothesis refinement
RESEARCH.md's hypothesis was *structurally correct* (kustomize-vs-helm drift) but *symptomatically wrong* (predicted CrashLoop; reality is timeout-during-cert-manager). Plan shape is unchanged. Plan 04 must verify that the post-fix `make test-int` completes well within the 300s inner timeout (not within ε of it) — recorded as a forward-looking surface in the artifact's Section 4.
