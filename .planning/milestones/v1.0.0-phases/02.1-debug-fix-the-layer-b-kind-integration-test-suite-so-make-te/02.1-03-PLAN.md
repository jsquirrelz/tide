<objective>
Add a single plaintext boot banner line to `cmd/credproxy/main.go` so the Layer B credproxy log-line test in `test/integration/kind/credproxy_test.go` (the 7th of 7 Layer B specs) passes.

Per D-01 in `02.1-BASELINE.md`: approach (a) — emit `fmt.Fprintf(os.Stdout, "credproxy listening on %s\n", listenAddr)` directly to stdout, immediately before `p.ListenAndServe(ctx)`, KEEPING the existing structured zap `log.Info("starting credential proxy", ...)` line. The structured log is for machine consumption (zap-behind-logr JSON shape); the plaintext banner is for human readability + test grep stability. This is a known pattern in K8s operators (etcd, kubelet, kube-proxy all emit human-readable boot banners alongside their structured logs).

This plan is independent of Plan 02 (different file). Both Plan 02 and Plan 03 land in Wave 2 in parallel. Output: A modified `cmd/credproxy/main.go` with one new `fmt.Fprintf(os.Stdout, ...)` line and a brief comment explaining why plaintext lives alongside structured logging. `go.mod` and `go.sum` are unchanged.
