<objective>
Fix defect #10b (same cross-namespace bug class as #11/#12): today `TaskReconciler.buildEnvelopeIn` calls `r.Deps.PromptReader.ReadPrompt` — the MANAGER reads the executor prompt off ITS `/workspaces` PVC, which is a different namespace-local volume than the project's PVC the subagent runs against. Purpose: the Manager-reads-PVC premise was always wrong cross-namespace; remove it for the prompt path.
Output: `EnvelopeIn.PromptPath`; in-pod read in `anthropic.Run`; Manager stops reading the prompt.
</objective>
