## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator → CRD annotations | Approve/reject annotations are the operator's write-back surface; RBAC governs who can write them, but the reconciler MUST consume them exactly once to prevent replay |
| Reconciler → child CRD list | BoundaryDetected lists child CRDs; the list must be RBAC-scoped to the controller's namespace (CTRL-04 / AUTH-03 — already enforced by existing controller RBAC) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-G1 | Spoofing / privilege | internal/gates/policy.go | mitigate | EvaluatePolicy reads only the CRD field; no env/annotation can override it. CEL validation on the CRD (CRD-03, already in api/v1alpha1) limits values to {auto, approve, pause}. |
| T-04-G2 | Tampering (replay-approval) | internal/gates/annotation.go | mitigate | ConsumeApprove returns a NEW map; reconciler patches the cleared map. After one-shot consume, re-triggering the gate requires a fresh annotation write — matches budget.ConsumeBypass pattern (Phase 2 D-D4). |
| T-04-G3 | Spoofing (wave-skip) | internal/gates/annotation.go (CheckWaveApprove) | mitigate | CheckWaveApprove keys the annotation off the integer wave index — annotation `approve-wave-3` does NOT approve wave 4. Test 4 asserts this. |
| T-04-W2 | DoS (boundary loop) | internal/gates/boundary.go | mitigate | BoundaryDetected returns false on empty child list and on any non-Succeeded child; reconciler must not loop on a false result (caller-responsibility, but the function is pure and idempotent). |
</threat_model>
