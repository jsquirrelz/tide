<objective>

Replace with `json.NewDecoder(...).Decode` (stdlib; stops at end of first JSON value, ignoring trailing prose) and add per-file isolation so a single bad file does not lose its valid siblings.

With the current hard-abort, this fails the entire dispatch and loses 27c + all valid children. The fix preserves valid siblings and surfaces a clear, retriable error naming the bad file.

Output: Modified `readChildCRDs` in `subagent.go` (json.Decoder + dec.More() + per-file isolation); updated and new tests in `childcrd_read_test.go`; prompt patch in `plan_planner.tmpl`.
