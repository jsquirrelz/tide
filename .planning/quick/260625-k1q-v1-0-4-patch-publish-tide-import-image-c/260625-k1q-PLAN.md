# Quick Task 260625-k1q

## Problem

`tide-import` (the dogfood-#2 cost-saving import engine) is referenced by the
chart (`charts/tide/values.yaml:200` → `ghcr.io/jsquirrelz/tide-import`) and has
a Dockerfile (`images/tide-import/Dockerfile`), but it is **missing from the
`build-images` matrix** in `.github/workflows/release.yaml`. So no published
release ever builds/pushes `ghcr.io/jsquirrelz/tide-import:<appVersion>`, and a
chart-based `helm install` ImagePullBackOffs the import feature. The
`version-consistency` gate cannot catch this — it checks chart version skew, not
matrix↔chart image coverage.

## Tasks

### Task 1 — Add tide-import to the release build-images matrix
- **files:** `.github/workflows/release.yaml`
- **action:** Append a matrix entry `- component: tide-import` /
  `dockerfile: images/tide-import/Dockerfile` / `context: .` after the
  `tide-reporter` entry. Fix the stale image-count comments: "all 6 component
  images" → "all 8 component images" and the three "6 matrix jobs" → "8 matrix
  jobs" references (now 8 components: controller, dashboard, stub-subagent,
  credproxy, push, claude-subagent, reporter, import).
- **verify:** `grep -cE '^[[:space:]]*- component:' .github/workflows/release.yaml`
  returns `8`; no remaining stale `\b6\b` matrix/image references.
- **done:** matrix builds tide-import at the release tag's stripped version.

### Task 2 — Guardrail: every chart-referenced ghcr image is in the release matrix
- **files:** `hack/scripts/verify-chart-images-published.sh` (new),
  `Makefile` (new `verify-chart-images-published` target),
  `.github/workflows/ci.yaml` + `.github/workflows/release.yaml` (wire the gate)
- **action:** Pure-bash script (mirror `verify-version-consistency.sh`, no yq):
  extract every `ghcr.io/jsquirrelz/<name>` referenced in
  `charts/tide/values.yaml`, extract every `- component: <name>` in the
  release.yaml build-images matrix, and FAIL listing any chart image absent from
  the matrix. Add a `.PHONY` Makefile target. Wire `make
  verify-chart-images-published` into the ci.yaml verify-gates job (next to
  verify-version-consistency) AND the release.yaml `helmify-verify` job
  (defense-in-depth at release time — this is the gate that would have caught
  the tide-import miss before publish).
- **verify:** `make verify-chart-images-published` exits 0 after Task 1; exits 1
  (naming tide-import) if the matrix entry is removed.
- **done:** the matrix↔chart image-coverage invariant is enforced in CI and at
  release.
