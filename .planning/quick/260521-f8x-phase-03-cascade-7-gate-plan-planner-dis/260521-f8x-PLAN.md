**Why dispatch must be gated, not the credproxy guard removed:**

The `opts.Project != nil` guard in jobspec.go:266-272 is correct defense-in-depth — it prevents a nil deref when constructing the EnvFrom list. The problem is that proceeding to dispatch with `project == nil` produces a Job that *runs but cannot succeed* (credproxy crashes on missing `ANTHROPIC_API_KEY`). Dispatch is single-shot (idempotent on AlreadyExists per line 309), so the first nil-Project Create permanently wedges the Plan planner for that Plan UID. The fix is to never create the Job until the Project is resolvable.

**Why requeue (not error) on the transient arm:**

Returning a real `error` from `reconcilePlannerDispatch` would surface as a reconcile failure in metrics and would still trigger controller-runtime's default exponential backoff. The intent is benign: "the cache hasn't caught up yet, try again in 1s." That's `ctrl.Result{RequeueAfter: 1*time.Second}, nil` — a clean requeue with no error. The `false` "handled" sentinel ensures the caller does NOT treat this branch as a committed dispatch.

**Why log levels split Info vs V(1):**

- Empty PhaseRef is a configuration error — operators need to see it in default-verbosity logs to diagnose mis-applied CRDs. Info level.
- Transient cache-miss is expected on the hot path immediately after a Plan is created (the Plan reconciler can fire before the Phase informer cache has caught up). It should NOT be Info — that would spam logs on every fresh Plan creation. V(1) (debug verbosity).

**Out of scope** (do NOT touch in this quick task):

- `phase_controller.go` lines 195-200 — Phase planner has the symmetric nil-Project race with a 2-hop walk (`resolveProject` instead of `resolveProjectForPlan`). Same fix shape applies. Tracked as "Cascade 7-bis" in the SUMMARY follow-up list.
- `milestone_controller.go` lines 240-244 — Milestone planner has an even more latent nil-deref risk (different pattern that defaults `project` to nil and then accesses `project.Spec.ProviderSecretRef` — would PANIC). Defensive follow-up tracked in SUMMARY.
- Modifying `resolveProjectForPlan` to return `(*Project, error)` — would touch 4 call sites; over-scoped for this cascade.

- Touching `charts/tide/values.yaml` — chart is FIXED contract per CLAUDE.md.
